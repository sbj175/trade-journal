# OPT-289: Roll-detection fixture follow-ups + RollChainModal copy refresh

## Why

OPT-281 closed with the framework + 2 fixtures shipped. The remaining work is mechanical drop-ins: write a fixture module per remaining spec branch and add a row to `ALL_FIXTURES` in `tests/integration/test_roll_detection_spec.py`. Each is small (~100 lines) and low-risk. Plus one UI sync.

## Fixture follow-ups (each one ≈ one PR-sized commit)

Each fixture should expose `SPEC_SECTION`, `DESCRIPTION`, `transactions()`, `expected()` per the protocol in `tests/fixtures/roll_same_exp_diff_strike.py`.

- **`roll_legged_out_and_back_in.py`** — OPT-280 USO case. Same-day legged-out via separate orders. Must link via signature match when chain overlap is absent.
- **`roll_plus_add.py`** — IBIT-class roll-plus-add. Closes-of-shape-X have fewer lots than opens-of-shape-X on the same day. The greedy match consumes the closes; leftover opens are new business, not part of the chain. Tests spec §2 "Quantity overflow" + §1.1 closest-strike pairing.
- **`roll_partition_5_into_3_plus_2.py`** — Spec §5.2 branching example. Five-lot source group splits across two new groups; each new group has `every-lot-paired-into-source` so both get `rolled_from = source`.
- **`roll_with_mid_life_adjustment.py`** — Spec §5.3. A source group has a leg that was rolled out mid-life; the source's closing event must not pair with the rolled-out leg. Lot-level pairing handles this automatically.
- **`non_roll_cross_direction.py`** — Spec §2. Closed long Put cannot pair with newly-opened short Put (sign-of-quantity differs). The lot stays unpaired.
- **`non_roll_dates_differ.py`** — Spec §1.1.3. Same-day rule. Close on day 1 + open on day 2 must NOT link.
- **`non_roll_different_account.py`** — Spec §1.1.1. Cross-account links are out of scope.
- **`non_roll_different_underlying.py`** — Spec §1.1.2. Different underlying must NOT pair.

(Phantom-strangle (OPT-261) was deliberately omitted — the lot-level model handles it naturally and the bug it described is gone. If a real-world example surfaces, add it then.)

## UI sync

The `RollChainModal` component has a "learn more" expandable section whose text was the user-facing summary of the older group-level chain model. After OPT-284 / OPT-287 / OPT-288, the underlying model is purely lot-level lineage. The copy should be regenerated or hand-aligned against `docs/roll-detection-spec.md`.

- File: `frontend/src/components/RollChainModal.vue`
- Spec source: `docs/roll-detection-spec.md` §1 (what a roll is), §3 (adjustment vs roll), §5 (multi-roll dynamics)
- Keep it short — the modal is a quick reference, not a full explanation

## How to work this

These are independent units. Pick whichever one's next to come up in real data — that gives the fixture a real-world worked example to base on. Or batch a few in a single session if the energy is there. Either way, each fixture is a drop-in addition with no framework changes needed.
