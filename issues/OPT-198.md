# OPT-198: Background sync with account selection and progress widget

Redesign the initial sync and onboarding flow to be non-blocking with progressive data loading.

## Current Problem

* Initial sync blocks the UI for 3+ minutes with a spinner
* All accounts are imported regardless of user preference
* No progress indication beyond "Importing transactions..."
* Risk of HTTP timeout on slow connections

## Desired UX (inspired by OptionIncome)

### 1\. Account Selection

After OAuth connect, show the user's Tastytrade accounts and let them choose which to import:

* Display account number, type (Individual, Roth IRA, etc.), and nickname
* Checkbox to include/exclude each account
* Skip accounts the user doesn't care about (reduces sync time)

### 2\. Background Sync

* Kick off the import as a background task (asyncio task or task queue)
* Return immediately — user can browse the app while sync runs
* Store sync progress/status in the database (per account: pending, syncing, complete, error)

### 3\. Progress Widget

* Persistent banner or toast component visible across all pages during sync
* Shows per-account progress: "Importing 5WZ27378... 847 transactions"
* Updates via WebSocket (preferred) or polling
* Dismissible once complete

### 4\. Progressive Loading

* Pages start showing data as it arrives (account by account)
* Positions page shows imported accounts immediately, remaining accounts show "syncing..."
* Ledger/Reports populate incrementally as the pipeline processes each account

## Technical Considerations

* Backend: background asyncio task with status written to DB or in-memory store
* Frontend: WebSocket progress updates (we already have `/ws/quotes` infrastructure)
* Pipeline: process each account independently so partial results are available early
* Error handling: per-account errors don't block other accounts (already partially handled — `5WZ28644` errors don't stop the others)

## Incremental Sync

The same pattern should apply to regular syncs (not just initial) — sync runs in background, user sees a subtle progress indicator, pages update as new data arrives.
