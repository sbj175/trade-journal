# OPT-267: Remove the "Partially rolled" ⓘ indicator — redundant after OPT-263

## Why

The ⓘ "Partially rolled" hint on the Ledger row header was a best-effort signal from the old flat-lot-table era: it tried to warn users that the single lot table mashed together legs from multiple lifecycle events. [OPT-263](https://linear.app/optionedge/issue/OPT-263/rolled-position-groups-display-is-confusing-strategy-label-rolls) made that staging explicit (labeled Opening / Roll / Closing sections with per-section dates and net credit/debit), so the hint is now redundant — the staged timeline shows directly which legs came from the original opening versus a roll.

It's also wired to a fragile `opening_order_id` heuristic (`src/routers/ledger.py:180-184`): if a single logical roll happens to land on the broker as two order_ids, the flag fires even when the position isn't really in a split-rolled state. The walk-and-balance timeline ([OPT-263](https://linear.app/optionedge/issue/OPT-263/rolled-position-groups-display-is-confusing-strategy-label-rolls)) doesn't depend on order_id grouping and reports the reality cleanly.

**Verified**: zero groups in production data currently have `partially_rolled=true`. The flag only fires for currently-open groups with multiple distinct `opening_order_id`s among the open lots — in practice, rolls close all old legs and open all new legs in one pass, so the condition never triggers.

## Scope

* `src/routers/ledger.py:180-184` — drop the `partially_rolled` computation and the field from the response
* `frontend/src/components/LedgerDesktopRow.vue` — remove the ⓘ `<span>` (around line 49-51)
* `frontend/src/components/LedgerMobileCard.vue` — remove the ⓘ `<span>` (around line 45)

## Non-goals

* Not removing `opening_order_id` / `closing_order_id` from lots — those remain in use for order assembly, fees lookup, and audit trail.
* Not changing roll detection/counting — [OPT-263](https://linear.app/optionedge/issue/OPT-263/rolled-position-groups-display-is-confusing-strategy-label-rolls) walk-and-balance already owns that.

## Acceptance

* `/api/ledger` response no longer includes `partially_rolled`.
* Ledger row header never shows the ⓘ hint.
* No visible difference for any existing group (since the flag was false for all of them).
* Frontend and backend tests still pass.
