# OPT-124: Remove positions_inventory table and PositionInventoryManager

## Summary

The OPT-121 redesign specified that `positions_inventory` should be eliminated — lots are the only position tracking system. The table and `PositionInventoryManager` still exist and are actively used for backward compatibility.

## Current usage

* `PositionInventoryManager` is initialized as a singleton in `src/dependencies.py`
* Passed to `orchestrator.reprocess()` and `position_ledger.process_lots()`
* `process_lots()` calls `position_manager.update_position_from_transaction()` for every transaction
* `position_manager.clear_all_positions()` called at start of reprocessing

## What to do

1. Audit all code paths that read from `positions_inventory` — replace with lot-based queries
2. Remove `position_manager.update_position_from_transaction()` calls from `position_ledger.py`
3. Remove `PositionInventoryManager` from dependencies and pipeline
4. Drop the `positions_inventory` table (Alembic migration)
5. Delete `src/models/position_inventory.py`

## Risk

Low — the table is being kept in sync but lots are already the source of truth. Need to verify nothing in the frontend or API reads directly from `positions_inventory`.

## Depends on

Spun out of [OPT-121](https://linear.app/optionedge/issue/OPT-121).
