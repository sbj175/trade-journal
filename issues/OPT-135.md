# OPT-135: Implement tagging on Position Groups (same level as Strategy)


## Comments

### 2026-02-27 — Steve Johnson

* Tags should be attached to position groups and allow for 0 to many in cardinality.
* Position Groups to Tags should be a many-to-many relationship such that a tag that says "Bullish" would only exist once in the tags table.
* Filtering on the Positions page and Ledger page should include filtering by 1 or more tags
* When filtering by tag, matching should be for ANY tag in the filter

---

### 2026-02-27 — Steve Johnson

Implemented and merged to `main` in commit `6d41d88`.

**What was done:**

**Backend:**
- New `Tag` and `PositionGroupTag` models in `models.py` (user-scoped with unique constraints)
- Alembic migration `add_tags_009` creating both tables
- CRUD endpoints in `src/routers/tags.py`: GET/POST/PUT/DELETE `/api/tags`
- Group-tag association endpoints in `ledger.py`: POST/DELETE `/api/ledger/groups/{group_id}/tags`
- POST supports both `{tag_id}` (existing) and `{name}` (find-or-create) modes
- Tags batch-loaded and included in both `/api/open-chains` and `/api/ledger` responses

**Frontend (Positions + Ledger):**
- Tag chips displayed below strategy/roll badges with per-tag color
- `×` button on each chip to remove tag
- `+` button opens autocomplete popover
- Popover: type to filter existing tags, Enter to create new, click to select
- Tags state loaded on init, refreshed after creates
