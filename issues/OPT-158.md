# OPT-158: Show import summary before redirect to Positions

After the onboarding import flow completes, the app immediately redirects to `/positions` — the user never sees how many transactions were imported. Non-onboarding imports already show a toast, but onboarding skips it entirely.

**Changes:**

* Add a success modal after onboarding import showing transaction/order/chain counts
* Fix field name mismatch in non-onboarding toast (`orders_saved` → `orders_assembled`, `chains_saved` → `chains_derived`)
* User clicks "Go to Positions" button to proceed
