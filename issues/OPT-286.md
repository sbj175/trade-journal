# OPT-286: OPT-286: Ledger fee display — closing fees double-counted, combined-fill open fees lost

## Symptom

Surfaced in the Mar 6 IBIT 39C/Mar-9 group on the Ledger after [OPT-285](https://linear.app/optionedge/issue/OPT-285/opt-285-same-shape-opens-at-same-expiration-merge-into-one-group)'s same-shape merge. Two display bugs in `src/routers/ledger.py`:

### Bug 1: closing fee double-counted

When one BTC transaction closes multiple lots (the 80-contract buy-back closed both lot 29558 (qty=69) and lot 29559 (qty=11) under the same `closing_transaction_id` = 423940927, fee = $9.84), the router does `fees_by_txn.get(c.closing_transaction_id, 0)` for *each* closing and attributes the *full* transaction fee to *every* lot that closing touched. Result: $9.84 appears against the 69-leg AND $9.84 against the 11-leg — total displayed $19.68 vs actual $9.84.

### Bug 2: combined-fill open fees lost

When `lot_manager` combines multiple same-price open fills into one lot (e.g., lot 29558 represents three fills of 33+16+20 contracts at $0.48), the lot's `transaction_id` is a comma-separated string ("423940238,423940235,423939568"). `fees_by_txn.get("423940238,423940235,423939568", 0)` looks the whole string up as one key and returns 0. Result: the 69-contract open shows $0 fees instead of $4.169 + $2.023 + $2.526 = $8.72.

## Fix

In `src/routers/ledger.py`:

1. **Apportion closing fees by quantity_closed.** Pre-compute total quantity closed per `closing_transaction_id` across all lot_closings; then for each closing record, `closing_fees = txn_fee × this.quantity_closed / total_qty_for_txn`. The 69-leg gets $9.84 × 69/80 = $8.49; the 11-leg gets $1.35; sum = $9.84.
2. **Decompose combined-fill transaction_id.** When `lot.transaction_id` contains commas, split, look up each component, and sum. Apply both when adding to `all_txn_ids` and at fee-lookup time.

## Tests

Two unit tests in a new `test_ledger_fees.py` (or in `test_ledger_router.py` if it already exists):

* One closing transaction closing multiple lots → fee apportioned by quantity, sums to txn total.
* Lot with comma-separated transaction_id → open fee = sum of per-component fees.

## Production data

Display-only change. `lot_closings.realized_pnl` and chain summaries don't depend on fee display, so no data refresh needed.
