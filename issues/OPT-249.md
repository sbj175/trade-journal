# OPT-249: Landing: keyword-targeted H2 copy for SEO

The hero H1 is brand-voice ("The options trade journal that actually understands options") which reads well but doesn't contain the exact-match phrases people search for.

## Plan

Add keyword-targeted copy via H2s and body text throughout the page without breaking the editorial tone. Target queries:

* "tastytrade trade journal"
* "options trade journal"
* "options roll tracker"
* "iron condor tracker"
* "options P&L tracker"
* "tastytrade trade history analysis"

## Concrete suggestions

* Problem section H2: "Options trade journals weren't built for multi-leg traders" (adds "options trade journal" + "multi-leg")
* Solution section H2: "An options trade journal for Tastytrade, automatically organized" (adds "options trade journal for Tastytrade")
* Features section H2: keep as-is, but add feature row copy like "Roll tracker for options" and "Iron Condor P&L calculator"
* Add an FAQ section near the bottom targeting long-tail queries:
  * "Does OptionLedger work with Tastytrade?"
  * "Can it track rolls on multi-leg options strategies?"
  * "How is this different from TradesViz / TradeZella / TraderSync?"

## Why

Competitor names in FAQ copy are gold — people search "tastytrade journal vs tradesviz" and you can rank for those queries by simply addressing them. FAQ schema (JSON-LD `FAQPage`) can also earn rich-result accordions in SERPs.

## Notes

* Don't stuff keywords unnaturally — Google penalizes it
* One query per heading is plenty
* Keep the editorial broadsheet voice; augment, don't rewrite
