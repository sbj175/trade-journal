# OPT-277: Remove unreachable Cash Secured Put strategy label

## Decision

The `"Cash Secured Put"` label is unreachable from the recognizer (single short puts always go through `match_single` → `"Short Put"`; the option-only-combo path in `recognizer._try_match` is gated on `len(options) == 3` for Jade Lizard). Beyond being dead code, the user prefers `"Short Put"` semantically — whether a put is cash-secured is an account-config concern, not a structural property of the position.

## Changes

* `src/pipeline/strategy_engine/patterns_combo.py` — remove the 1-leg short put branch in `_match_option_only_combo`
* `src/pipeline/strategy_engine/constants.py` — remove the `"Cash Secured Put"` entry from `STRATEGIES`
* `src/database/db_manager.py` and `src/auth/user_provisioning.py` — remove `"Cash Secured Put"` from the seeded strategy_targets list
* `frontend/src/lib/constants.js` and `frontend/src/composables/useSettingsTargets.js` — remove from the credit-strategies list
* `frontend/src/pages/components/index.vue` — design showcase mock: replace with `"Short Put"`
* `tests/unit/test_strategy_engine.py` — rename `test_cash_secured_put` to reflect the actual assertion ("lone short put returns Short Put")

## Migration note

Existing users with `"Cash Secured Put"` rows in their `strategy_targets` table will still see those rows in Settings UI but the recognizer will never match them. Not migrating today — low impact, low frequency.

## Tests

`test_strategy_engine.py::TestSingleLeg::test_short_put` already asserts the right label. The renamed lone-short-put test reinforces the invariant. Full suite: 174+ passing.
