# OPT-154: Add data disclosure consent screen and privacy page

## Context

Tastytrade API Terms of Service (Other Terms #2) requires third-party integrators to "clearly disclose to any customers of Client if any data, including but not limited to, trade, position, financial or Client information is extracted, obtained, or pulled from the APIs and stored on their systems for any reason."

This disclosure is also needed to complete Exhibit A (question 3: how will you disclose data practices to users). See OPT-153 for the full ToS compliance review.

## Deliverables

### 1\. Consent screen on Tastytrade OAuth connection flow

Add a disclosure summary to the Settings page, shown before the user clicks "Connect to Tastytrade." The user must acknowledge before proceeding.

**Location:** `frontend/src/pages/settings/App.vue` — in the Tastytrade connection section

**Content to display:**

> **What OptionLedger accesses from your Tastytrade account:**
>
> * Account numbers and balances
> * All historical transactions (trades, assignments, expirations)
> * Current open positions
> * Real-time market quotes for your holdings
>
> **What is stored on our servers:**
>
> * Your encrypted Tastytrade connection token (never your password)
> * Transaction history, positions, and account balances
> * Cached market quotes (refreshed periodically, not persisted long-term)
> * User-created notes, tags, and settings
>
> **What is NOT stored or shared:**
>
> * Your Tastytrade password
> * Your data is never shared with third parties
> * Market data is never redistributed outside your account

**UX:** Checkbox or "I understand" acknowledgment before the Connect button becomes active. Store acknowledgment in user preferences or local storage so it's only shown once.

### 2\. Privacy / Data Practices page

A standalone page at `/privacy` with the full data disclosure. This is the page referenced when completing Exhibit A for Tastytrade.

**Content should cover:**

* What data is collected from Tastytrade (accounts, transactions, positions, quotes)
* What data is stored and where (encrypted at rest, per-user isolation)
* Data retention practices
* What is NOT collected (passwords, personal financial info beyond trading)
* No data sharing with third parties
* No redistribution of market data
* How users can delete their data (Settings page or account deletion)
* Contact information

**Location:** New route `/privacy`, linked from:

* Footer on all pages
* Settings page (near the Tastytrade connection section)
* The consent screen ("See our full data practices")

## Implementation Notes

* The privacy page can be a simple static-content Vue SFC (no API calls needed)
* Add a new route in `src/routers/pages.py` for `/privacy`
* The consent screen is a UI-only change to the Settings page
* No backend changes required for either deliverable
* Content should be reviewed before SaaS launch (may need legal review)
