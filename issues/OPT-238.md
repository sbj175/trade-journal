# OPT-238: Use account opened_at date to determine initial sync import range

Currently the sync uses hardcoded day windows (365 days for first sync, 730 days for initial sync) to determine how far back to fetch transactions. The Tastytrade SDK's Account model has an `opened_at` field that tells us when the account was opened.

**Changes:**

* Add `opened_at` (DateTime) column to the Account model
* Extract `opened_at` from the SDK in `get_all_accounts()`
* Store it via `save_account()` in db_manager
* Use the earliest account `opened_at` as the default start date for initial sync (instead of hardcoded 730 days)
* For regular first sync, use `opened_at` instead of hardcoded 365 days
* Remove MAX_DAYS_BACK cap since the real account date is authoritative
