# OPT-264: Both-legs-ITM call spread expiration creates phantom equity lot

## Problem

When a vertical call spread (or any spread) settles at expiration with **both legs ITM**, the broker delivers one BTO equity transaction (from the exercised long call) and one STC equity transaction (from the assigned short call) at the same timestamp. These cash-settle net-zero shares with a realized stock P&L equal to `(short_strike − long_strike) × contracts × 100`.

The pipeline currently handles the option side correctly (closes both option lots with derivation_type ASSIGNMENT/EXERCISE) but **mishandles the equity side**: it creates a long equity lot from the BTO and never finds the offsetting STC, leaving a phantom open equity position and an unbooked stock-leg P&L.

## Repro

* User: [sbj175@gmail.com](<mailto:sbj175@gmail.com>), Individual Margin account (5WZ27378)
* Symbol: HUT
* 2025-05-05: Opened Bear Call Spread — short 14C / long 15C, expiration 2025-06-20 (+0.29 credit)
* 2025-06-20: HUT closed above 15, both legs ITM. Broker delivered four rows at 2025-06-20 21:00 UTC:
  * Exercise on 15C long → BTO 100 HUT @ $15
  * Assignment on 14C short → STC 100 HUT @ $14
* **Observed**: Positions page shows 100 HUT shares owned (phantom). Options-only P&L = +$29. Equity leg P&L never booked.
* **Expected**: zero HUT shares, realized P&L for the whole trade = -$71 (spread's max loss).

DB snapshot:

* `position_lots` row 10353: EQUITY, HUT, qty=100, remaining_quantity=100, entry_price=15.00, status=OPEN, derivation_type=NULL, derived_from_lot_id=NULL
* `lot_closings` has correct rows for option lots 10248/10249 (EXERCISE/ASSIGNMENT)
* No `lot_closings` row for lot 10353; the STC raw transaction exists but was never linked

## Consequences

1. **Phantom position** on the Positions page — user appears to hold 100 HUT shares they don't own
2. **P&L error of $100 per contract** on the trade — spread looks like +$29 profit but was actually -$71 loss. Affects Reports totals, strategy-performance breakdowns, and win-rate metrics.
3. **Account balance reconciliation** may be off by the missing equity P&L

## Root cause

`src/pipeline/position_ledger.py` processes equity-side settlement per-option-event:

* Line 212-221: `tx.is_exercise` / `tx.is_assignment` branch dispatch
* Line 385, 401, 548-554: `create_derived_lot(..., derivation_type=...)` always creates a new lot; no short-circuit for a simultaneous offsetting stock leg
* Line 642-699: `_find_matching_stock` pairs one stock transaction per option event using same-timestamp + same-underlying + same-qty matching, with a strike tiebreak routing BTO@15 → Exercise (long-call-side) and STC@14 → Assignment (short-call-side). Individually each match is correct.
* Line 573+: `_process_deferred_closing` retries a TO_CLOSE that found no open lot, but if the EXERCISE-derived lot ends up in a different group/chain than the ASSIGNMENT close attempts to FIFO-close, the close fails silently.

Net effect: the two handlers run independently and never recognize the offsetting pair.

## Where to fix

Two viable directions:

**A) Spread-settlement detector.** Before the option-event loop, scan the day's raw transactions for clusters with two option events at the same timestamp on the same underlying whose derived-equity txns are offsetting (equal quantity, BTO + STC). Treat as a net-zero settlement: close the option lots normally, book the stock P&L as `(STC_price − BTO_price) × qty`, do NOT create a lingering equity lot.

**B) Post-settlement netter.** Let the handlers create both lots (long from Exercise, immediate close from Assignment), but add a reconciliation pass that detects same-timestamp offsetting equity lots on the same underlying and collapses them into a single realized-P&L event.

**A** is more principled (never creates the phantom in the first place); **B** is easier to bolt on without reworking the inline handlers.

## Acceptance

* After reprocess, the HUT 2025-06-20 call-spread expiration produces zero open HUT equity lots
* The trade's total realized P&L equals `(short_strike − long_strike) × contracts × -100 + net_credit × 100` = -$71 for this case
* Reports show the correct total loss
* No regression for other ITM-at-expiration scenarios:
  * Only long leg ITM (long ITM + short OTM) → genuinely leaves long shares, unchanged
  * Only short leg ITM (short ITM + long OTM) → genuinely leaves short shares, unchanged
  * Neither leg ITM (both worthless) → no equity effect, unchanged
