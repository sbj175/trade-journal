# OPT-146: Ledger page position notes not saving reliably

**Problem:** Position-level notes on the Ledger page don't reliably persist. Deleted notes reappear after navigating to Ledger and back.

**Root causes:**

1. `loadNotes()` is not awaited in Ledger `init()` — race condition where async load overwrites in-memory edits
2. Save errors (HTTP 401/500) silently ignored — `.catch()` only handles network errors
3. `_migrateCommentKeys()` runs on every Positions page load — old `chain_*` keys in the DB are never deleted after migration, so every time positions load, the migration re-creates notes under `group_*` keys even if the user deleted them

**Fixes:**

* Await `loadNotes()` in Ledger init
* Check `response.ok` after save requests
* Delete old `chain_*` keys from DB after migration so they can't resurrect deleted notes
