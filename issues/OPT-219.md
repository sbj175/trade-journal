# OPT-219: Use TT account opened_at date for automatic import range

## Summary

The Tastytrade SDK provides `opened_at` on each Account object (e.g., `2025-03-21 17:30:40`). Use this as the default start date for importing transactions instead of asking the user to pick a date.

## Impact

* Eliminates the date picker from the Import Transactions flow
* Simplifies onboarding: enable accounts → import → done
* Could potentially fold the Import tab into the Accounts tab entirely
* Per-account import already exists (`POST /api/sync/account/{number}`) — just needs to use `opened_at` as the start date

## Available fields on TT Account object

* `opened_at` — when the account was opened (best for import start date)
* `created_at` — when the record was created (similar but slightly different)
* `closed_at` — null for active accounts
* `funding_date` — when first funded (could also be useful)

## Steps

1. Save `opened_at` to the Account model (new column or use existing fields)
2. Use it as default `start_date` in `/api/sync/account/{number}` and `/api/sync/initial`
3. Remove the date picker from the Import Trades tab
4. Consider merging Import tab functionality into the Accounts tab
