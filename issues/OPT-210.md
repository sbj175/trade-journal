# OPT-210: Ledger: Surface fees column in detail rows

## Context

Fee data (regulatory_fees, clearing_fees, commission) exists on `RawTransaction` but is not surfaced anywhere in the UI. Users need to see total fees per leg in the Ledger detail view.

## Requirements

1. **Backend**: Add fee aggregation to the `/api/ledger` endpoint — sum fees from related `RawTransaction` records for each lot/closing
2. **Frontend**: Add a "Fees" column to the Ledger detail row, positioned to the right of the Close Status column
3. Fees should aggregate correctly in `groupedOptionLegs()` when lots are consolidated by strike/disposition

## Fee columns on RawTransaction

* `regulatory_fees`
* `clearing_fees`
* `commission`
* `net_value`
* `is_estimated_fee`

## Display

* Show total fees (sum of regulatory + clearing + commission) per aggregated leg
* Negative values (costs) displayed in red
* Column header: "Fees"
