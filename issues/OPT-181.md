# OPT-181: New positions from sync not added to WebSocket price feed until page refresh

## Bug

After performing a sync from the Positions page, newly synced positions do not appear in the WebSocket price feed. Their live quotes/P&L do not update until the user either:

* Navigates away and back to the Positions page, or
* Does a full browser refresh

## Expected Behavior

After a sync completes and new positions are returned, the WebSocket subscription should automatically update to include symbols for any newly added positions so they receive live price data immediately.

## Likely Cause

The WebSocket symbol subscription list is probably built once on page mount and not refreshed after a sync operation completes. The sync response handler likely updates the position data in the UI but doesn't trigger a re-subscription to the WebSocket feed with the updated symbol list.

## Comments

### 2026-03-30 — Steve Johnson

Already fixed. `fetchPositions()` calls `requestLiveQuotes()` after sync completes (line 132 of `usePositionsData.js`). The flow is: sync → fetch updated chains → `applyFilters()` updates `filteredItems` with new symbols → `requestLiveQuotes()` resubscribes the WebSocket with the full updated symbol list → backend returns immediate quotes for all symbols.

Closing as resolved.
