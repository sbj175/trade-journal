# OPT-153: Tastytrade API Terms of Service — SaaS compliance review

## Summary

Full review of the [Tastytrade API Terms of Service](<https://assets.tastyworks.com/production/documents/open_api_terms_and_conditions.pdf>) (last updated May 17, 2023) for compliance with deploying OptionLedger as a multi-user SaaS product. The community SDK ([tastyware/tastytrade](<https://github.com/tastyware/tastytrade>)) is MIT-licensed with no restrictions.

## Current Architecture Compliance

Our per-user OAuth2 flow aligns well with the technical requirements:

| Requirement | Status | How |
| -- | -- | -- |
| No storing customer credentials | Compliant | OAuth2 refresh tokens (encrypted), no passwords stored |
| Per-user unique tokens | Compliant | `ConnectionManager` per-user pool, no token sharing |
| Use TT auth mechanisms | Compliant | OAuth2 authorization code flow via Tastytrade endpoints |
| No data redistribution to non-TT clients | Compliant | Each user must connect their own TT account |
| Closed Environment (users identifiable) | Compliant | Supabase Auth identifies every user |

## Key Terms That Affect SaaS Launch

### Permitted Purpose (Section 13)

> "your use of the API Connection to receive Data... such use so you can **build out your own value-add front end platform** or algorithmic trading systems."

Explicitly allows building a front-end platform on top of their API.

### Third-Party Integrator Rules (Other Terms, p.14)

1. **No storing customer credentials directly** — must use TT auth mechanisms (we do)
2. **Must clearly disclose to users** what trade, position, financial, or client data is extracted from the API and stored on our systems
3. **Autotrading permitted** for enrolled customers (not applicable — OptionLedger is read-only)
4. **Integrator liable for errant orders** from software malfunction (not applicable — read-only)
5. **Must not redistribute quotes or Data to non-TT clients** — must enforce per-user tokens so only TT clients access Data
6. **TT can terminate immediately** if they perceive regulatory or reputational risk

### Data Restrictions (Section 8)

* Data use limited to **trading and watchlist activity** — no bulk market data collection (8.3)
* **Cannot redistribute, publish, or make Data available to third parties** (8.4, 8.6, 8.11)
* **Derived data is still "Data"** if the original can be re-engineered from it (8.10) — cached quotes, P&L calculations may fall under this
* TT can **change data storage rules** at any time and we must comply (8.13)
* Data provided **"AS IS"** with no warranties (8.14)

### Software Ownership (Section 2.3.f)

> "any application or software that you use in conjunction with the API Connection will be a software or application that is **developed and owned by you** unless you have notified us of the nature of the software or application and received our prior written approval"

The community SDK (`tastyware/tastytrade`) is third-party open-source. We may need to notify Tastytrade and get written approval, or evaluate switching to their official SDK (`tastytrade/tastytrade-sdk-python`).

### Exhibit A (p.15)

A **fill-in-the-blank form** that must be completed declaring:

1. What data the app displays to users
2. What data is stored on our systems
3. How we disclose this to users

This strongly implies a **formal registration process** before operating as a third-party integrator.

### Other Notable Terms

* **Audit rights** (Section 9): TT can audit our systems once per year, records must be kept 2 years
* **No TT branding** (Section 2.12): Cannot hold ourselves out as representing Tastytrade
* **Throttle compliance** (Section 2.9): Must respect rate limits; persistent violation = termination
* **Monitoring consent** (Section 2.11): TT monitors API usage and we cannot interfere
* **Governing law**: State of Illinois, arbitration per Customer Agreement

## Action Items Before SaaS Launch

### Required

- [ ] **Contact Tastytrade** to register as a third-party integrator and complete Exhibit A
- [ ] **Disclose community SDK usage** (Section 2.3.f) and get written approval, or evaluate their official SDK
- [ ] **Add data disclosure / privacy notice** clearly stating what data OptionLedger extracts, stores, and displays (Other Terms #2)
- [ ] **Ensure non-TT users cannot access market data** — the app must gate quote/position data behind a connected TT account

### Recommended

- [ ] **Document our data storage practices** for potential audit (Section 9)
- [ ] **Add TT attribution disclaimer** to the app (we are not affiliated with or endorsed by Tastytrade)
- [ ] **Review rate limiting** to ensure compliance with throttle requirements
- [ ] **Evaluate official SDK** (`tastytrade-sdk-python`) as an alternative to the community SDK — may simplify the approval conversation
