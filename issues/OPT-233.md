# OPT-233: Inline chain P&L and improved roll indicator on Positions page

## Summary

When a position has been rolled, show chain-level P&L context inline in the position row — no click required. The row naturally grows taller to accommodate the extra info, only when rolls are present.

## Changes

1. **Inline chain P&L sub-row**: Add a secondary line below the main position row content showing chain net premium, realized P&L, and roll count. Only renders when `group.roll_chain` exists. Visually subordinate (smaller, muted text).
2. **Improved roll indicator**: Upgrade from the small blue chain icon to a more visible badge/pill so users actually notice rolled positions.

## Design Principle

Position data is actionable (drives mechanical decisions like "close at 50%"). Chain data is contextual (informs judgment). The primary row stays unchanged; chain info appears as a quiet secondary line.

## Context

Inspired by JTBD analysis — the chain P&L currently requires clicking through to the roll chain modal to see. Users making position management decisions are missing context that could influence their judgment.

No API changes needed — `group.roll_chain` already has `roll_count`, `cumulative_premium`, and `cumulative_realized_pnl` on the frontend.
