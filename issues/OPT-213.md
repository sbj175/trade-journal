# OPT-213: Market status indicator with expandable session details

## Summary

Replace the current "LIVE" indicator on the Positions page with a market-aware status that shows connection state AND market conditions. Make it clickable to expand and show detailed session info from the Tastytrade API.

## Current Behavior

* Shows "LIVE" with a green dot when WebSocket is connected to Tastytrade
* No indication of whether the market is actually open or closed
* Misleading — "LIVE" implies quotes are streaming even when markets are closed

## Proposed Behavior

### Collapsed State (always visible)

* **Connected + Market Open**: `● Market Open` (green)
* **Connected + Pre-Market**: `● Pre-Market` (yellow)
* **Connected + Extended Hours**: `● Extended` (yellow)
* **Connected + Market Closed**: `● Market Closed` (gray/muted)
* **Disconnected**: `○ Disconnected` (red)

### Expanded State (click to toggle)

Show a dropdown/popover with session details:

* Current status per exchange (NYSE, CME, CFE)
* Market open/close times for current session
* Next session open time (useful when closed)
* Extended hours close time
* Last quote update timestamp

## Implementation

### Backend

1. Add endpoint `GET /api/market-status` that calls `get_market_sessions()` from tastytrade SDK
2. Return status for relevant exchanges (NYSE for equities, CFE for options)
3. Include `get_market_holidays()` data for holiday awareness
4. Cache response (market sessions don't change frequently — 60s TTL)

### Frontend

5. Replace the "LIVE" indicator in `PositionsToolbar.vue` with the new market status component
6. Clickable to expand a popover showing per-exchange session details
7. Poll `/api/market-status` on mount and periodically (every 60s)

### SDK Available

* `tastytrade.market_sessions.get_market_sessions(session, exchanges)` — already in SDK
* `MarketStatus` enum: `OPEN`, `CLOSED`, `PRE_MARKET`, `EXTENDED`
* `MarketSession` dataclass: `status`, `open_at`, `close_at`, `close_at_ext`, `next_session`
* Exchanges: NYSE, CME, CFE, SMALLS
* Currently zero references in our code — needs to be wired up
