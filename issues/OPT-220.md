# OPT-220: Roll chain premium computed in two places — consolidate to single source

Roll chain net premium is computed independently in two places with duplicated logic:

1. `src/pipeline/roll_chain_summary.py` — materialized during pipeline
2. `src/routers/ledger.py` `/api/ledger/group-roll-chain` — computed on-the-fly for modal

Both had the same abs() bug (fixed separately). Consolidate to a single computation — the modal should read from the materialized table instead of recomputing.
