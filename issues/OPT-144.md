# OPT-144: Shared quote broadcaster for multi-user WebSocket scaling

## Problem

The current WebSocket quote streaming (`/ws/quotes`) has each connected client trigger its own `get_quotes()` call to Tastytrade every 5 seconds. With N concurrent users, that's N API calls per cycle — which will hit Tastytrade rate limits well before 100 users.

## Current Architecture

* Each WebSocket connection runs its own `send_updates()` loop (5s interval)
* Each loop calls `client.get_quotes(subscribed_symbols)` independently
* Client-side quote cache in `TastytradeClient` is 30 seconds, but each user has their own client instance

## Proposed Solution

**Shared quote broadcaster pattern:**

1. A single background task collects all subscribed symbols across all connected WebSocket clients
2. One `get_quotes()` call per cycle fetches all needed symbols
3. Results are broadcast to each WebSocket filtered by their subscriptions
4. 100 users watching the same 50 symbols = 1 API call per cycle, not 100

**Redis pub/sub** can serve as the broadcast backbone:

* Enables multiple app instances (horizontal scaling) to share a single quote stream
* One instance designated as the fetcher, others subscribe via Redis
* Already available in the Docker Compose stack (OPT-143)

## Key Considerations

* Interval should be configurable via env var (`QUOTE_INTERVAL_SECONDS`)
* Graceful degradation if Redis is unavailable (fall back to per-connection fetching)
* Symbol deduplication across all subscribers
* Clean removal of symbols when clients disconnect
