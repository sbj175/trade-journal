# OPT-218: WebSocket reconnect spams 403 after session expires

When the WebSocket connection drops (server restart, token expiry), the client retries every 5 seconds indefinitely. Each attempt gets a 403 Forbidden, spamming the server logs.

The reconnect logic in `usePositionsData.js` (`ws.onclose` handler) should:

1. Stop retrying after N failures (e.g., 3-5 attempts)
2. Or check if the auth token is still valid before reconnecting
3. Or redirect to login if the token is expired

Current behavior: infinite 403 loop every 5 seconds until the user manually refreshes.
