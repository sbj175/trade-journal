# OPT-298: Settings — General section + colorblind-friendly palette toggle (Okabe-Ito)

The light theme's `--tv-green` (`#507252` dusty sage) and `--tv-red` (`#9e5050` muted terracotta) both sit in the red-green confusable band at similar luminance, which collapses entirely for users with deuteranopia/protanopia. Verified live with the user: even side-by-side it's hard to tell up from down on the light theme; the dark theme is workable but still subtle.

Add a user-controlled toggle that swaps the two accent tokens to **Okabe-Ito CVD-safe values** which shift away from the red-green axis (green → cyan-leaning teal, red → orange-leaning vermillion) so dichromats can perceive the contrast on the blue-yellow axis instead. Keep the financial green=up / red=down convention.

## Scope

1. **`frontend/src/styles/main.css`** — add two new blocks that override only `--tv-green` and `--tv-red`:
   ```css
   :root.cvd {                /* dark + cvd */
     --tv-green: 0 208 149;   /* #00d095 */
     --tv-red:   255 111 60;  /* #ff6f3c */
   }
   :root.light.cvd {          /* light + cvd */
     --tv-green: 0 117 79;    /* #00754f */
     --tv-red:   194 64 28;   /* #c2401c */
   }
   ```
2. **`static/index.html`** — extend the pre-mount theme init to also restore `cvd` from localStorage, so the page doesn't flash with the standard palette before Vue mounts.
3. **`frontend/src/components/SettingsGeneral.vue`** — new component containing the toggle (and room for future "general" settings).
4. **`frontend/src/pages/settings/index.vue`** — register a new `general` tab in the nav, place it first.

Persistence pattern mirrors the existing theme toggle in `NavBar.vue`: write `'true'` / `'false'` to `localStorage.colorblindPalette`, add/remove `.cvd` class on `document.documentElement`.

## Out of scope

- Reorganizing or renaming other Settings sections — tracked separately in OPT-299.
- A high-contrast mode beyond the green/red shift.
- Changing chart libraries (ApexCharts on the Risk page); those use their own palettes and would be a follow-up if needed.

## Comments

### 2026-05-19 — shipped

Added `SettingsGeneral.vue` with the colorblind-friendly toggle, registered as the first tab in the Settings page. CSS variables `--tv-green` / `--tv-red` swap globally via `.cvd` class on `<html>`, persisted in `localStorage.colorblindPalette`, and applied pre-mount by the boot script in `static/index.html` so there's no flash of the standard palette on reload. Description block uses `max-w-prose` so the explanatory copy wraps at a readable line length even on wide screens.

The override is global — every place in the app that uses the `tv-green`/`tv-red` tokens (live prices, P&L values, chain B/E, badges, notifications) follows the palette automatically. ApexCharts on the Risk page have their own series colors and would need a separate follow-up if chart colors should also follow the palette.
