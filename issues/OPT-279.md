# OPT-279: Same-expiration rolls create new groups instead of staying in one

## Bug

A position rolled to a new strike pair at the **same expiration** (e.g., a Bull Call Spread 230/250 → 240/260, both Mar 20) created a fresh position_group linked via `rolled_from` instead of staying in the original group with an incrementing roll counter. The same-exp roll badge in the Ledger UI ("N rolls") therefore showed nothing because no group ever accumulated multiple roll generations.

## Real-world example (JNJ in two accounts)

```
gid 8535 (230/250, Mar 20)  Jan 28 → Feb 17  rf=None
gid 8577 (240/260, Mar 20)  Feb 17 → Feb 27  rf=8535  ← same-exp roll, should have stayed in 8535
gid 8598 (250/270, Apr 17)  Feb 27 → Mar 5   rf=8577  ← diff-exp roll, correctly its own group
```

## Root cause

[OPT-270](https://linear.app/optionedge/issue/OPT-270/roll-chain-breaks-when-two-independent-positions-on-the-same) / [OPT-274](https://linear.app/optionedge/issue/OPT-274/tighten-rule-1-group-routing-require-chain-overlap-with-open-lots) added `_is_group_open(gk)` guards to Rule 1 (and the chain-aware Rule 0) so closed-then-reopened-with-different-chain trades don't merge ([OPT-261](https://linear.app/optionedge/issue/OPT-261/position-groups-dont-reset-when-all-lots-close-before-new-lots-open)). But for a same-exp roll, the close and the open happen in one ROLLING order — Stage 5 sees the candidate group as "fully closed" by then, and Rule 0 falls through, creating a new group.

## Fix

Add a `same_exp_continuation` alternative to Rule 0 in `_route_lot_to_group`. When the new lot's `chain_id` is mapped to an existing group AND at least one existing lot in that group shares the new lot's `expiration`, route to the existing group regardless of open/closed status. This:

* Restores same-exp roll = stays in one group (Ledger badge shows count)
* Diff-exp roll = still creates new group + rolled_from link (no shared expiration ⇒ no match)
* [OPT-261](https://linear.app/optionedge/issue/OPT-261/position-groups-dont-reset-when-all-lots-close-before-new-lots-open) still fixed (different chain ⇒ Rule 0 doesn't fire at all)
* IBIT parallel-rung rolls unchanged (different chains)
* Cross-order Iron Condor unchanged (covered by Rule 1 structural complementarity)

## Tests
