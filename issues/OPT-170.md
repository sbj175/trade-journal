# OPT-170: Ledger group merge suggestions: detect potential multi-group strategies

## Summary

Instead of automatically merging groups that might form a multi-order strategy (Covered Call, Iron Condor, etc.), detect potential relationships and surface them as suggestions on the Ledger page. The user decides whether to merge.

## Why suggestions instead of auto-merge

Shares and short calls for the same underlying are often in different chains (the share purchase and the call sale are independent orders with no lot-closing relationship). Auto-merging by (account, underlying) is fragile — it can't reliably determine temporal overlap without heuristics. Letting the user confirm avoids false merges and respects their knowledge of their own trading intent.

## How it works

### Detection algorithm

For each (account, underlying) with multiple open groups:

1. Collect all open groups
2. Try pairwise (and n-wise) combinations of groups
3. For each combination, merge the lots speculatively and run the strategy engine
4. If the engine returns a more specific strategy than the individual groups would produce (e.g., "Iron Condor" instead of two separate spreads), that's a suggestion

This is general-purpose — the strategy engine handles pattern matching, so any new strategy added to the engine automatically becomes detectable. No hardcoded relationship rules needed.

### Examples of detectable suggestions

| Groups found separately | Combined strategy |
| -- | -- |
| Shares + Short Call | Covered Call |
| Bull Put Spread + Bear Call Spread (same expiry) | Iron Condor |
| Short Put + Bear Call Spread | Jade Lizard |
| Shares + Short Call + Long Put | Collar |
| Shares + Short Put + Short Call | Covered Strangle |
| 2 separate vertical spreads (same type, 3 strikes) | Butterfly |

### Backend

**New endpoint:** `GET /api/ledger/suggestions`

Query params: `account_number` (optional)

Returns:

```json
{
  "suggestions": [
    {
      "id": "suggestion-uuid",
      "type": "merge",
      "resulting_strategy": "Covered Call",
      "underlying": "IBIT",
      "account_number": "5WZ26959",
      "groups": [
        { "group_id": "abc", "strategy_label": "Shares", "status": "OPEN" },
        { "group_id": "def", "strategy_label": "Short Call", "status": "OPEN" }
      ],
      "description": "IBIT: Shares + Short Call could be a Covered Call"
    }
  ]
}
```

**Implementation:**

* Load all OPEN position groups for the account
* Group by (account, underlying)
* For each underlying with 2+ groups, speculatively combine lots and run `recognize()`
* Filter to suggestions where the combined strategy is more specific than individual labels
* Optionally: also check closed groups within a recent time window for historical suggestions

**Accepting a suggestion:** Frontend calls the existing `POST /api/ledger/move-lots` endpoint to move lots from one group into the other. The strategy re-eval in `_refresh_group_status()` (OPT-168) automatically updates the label.

**Dismissing a suggestion:** Store dismissal in localStorage (or a DB table if we want persistence across browsers). Dismissed suggestions don't reappear unless the underlying data changes.

### Frontend (Ledger page)

* Fetch suggestions on page load (after groups load)
* Display as a collapsible banner or inline card above the group list
* Each suggestion shows: underlying, current group labels, proposed combined strategy
* Two actions: **Merge** (triggers lot move) and **Dismiss** (hides suggestion)
* Badge on the suggestion bar showing count (e.g., "3 suggestions")

### Scope considerations

* **Open groups only** for initial version — closed groups are historical and less actionable
* **Same expiration preference** — for Iron Condors and similar, the suggestion should only fire when expirations match (the strategy engine already checks this)
* **Performance** — pairwise combinations of N groups is O(N²) per underlying, which is fine for typical portfolio sizes (< 20 open groups per underlying)

## Depends on

* **OPT-168** — strategy re-eval on lot moves must work so accepting a suggestion auto-relabels the merged group

## Future extensions

* Suggest splits (e.g., "This Custom 4-leg could be split into Bull Put Spread + Bear Call Spread")
* Suggest across closed groups for historical cleanup
* Wheel progression detection (Short Put → Assignment → Covered Call sequence)
