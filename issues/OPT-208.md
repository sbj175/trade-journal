# OPT-208: Extract page logic into composables for all pages

## Context

The Ledger page (`frontend/src/pages/ledger/App.vue`) has all logic embedded in a single `<script setup>` block. As features grow (OPT-207 close status, lot grouping, etc.), the file is getting large and harder to navigate.

## Proposed Structure

Extract page-specific logic into composables co-located with the page:

```
frontend/src/pages/ledger/
├── App.vue              (template + minimal glue)
├── useLedgerLots.js     (lot grouping, sorting, close status, equity aggregation)
├── useLedgerGroups.js   (group filtering, sorting, CRUD operations)
└── useLedgerState.js    (shared reactive state, filters, account selection)
```

Each composable exports a function that takes reactive data as input and returns computed values and methods. The `.vue` file stays focused on the template and wiring composables together.

## Functions to extract

**useLedgerLots.js**: `groupedOptionLegs`, `_addToLegMap`, `_buildCloseStatus`, `sortedOptionLots`, `sortedLots`, `lotCloseSummary`, `lotCloseDate`, `openEquityLots`, `equityAggregate`, `groupInitialPremium`

**useLedgerGroups.js**: group sort/filter logic, `saveGroupLabel`, `deleteGroup`, `createGroup`, `moveLots`, roll chain helpers

**useLedgerState.js**: `groups`, `filteredGroups`, filter refs, sort state, expanded state, account/underlying selection

## Notes

* Pure refactor — no behavior changes
* Should apply the same pattern to other large pages (Positions, Reports) eventually
* Consider after OPT-207 is complete since that adds more lot logic
