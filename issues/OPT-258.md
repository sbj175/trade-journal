## Follow-up: Server-side proxy + filesystem cache for LogoKit logos

OPT-258 added the logos with an in-memory per-page dedupe (`TickerLogo.vue` + `useTickerLogo.js`) so a Ledger view with many rows of the same underlying only fires one LogoKit request per ticker per page-view. That dedupe is still **per-session** — every page reload re-fetches, every user re-fetches, and we have no control over LogoKit's cache headers (which is what made the client-side dedupe necessary in the first place).

A proper follow-up would cap LogoKit usage at **one request per unique ticker per server lifetime**, across all users and reloads. Capturing the design here so we can pick it up when a Linear issue slot opens.

### Approach

Add a FastAPI endpoint (e.g. `GET /api/ticker-logos/{symbol}.png` or `/static/ticker-logos/{symbol}.png`) that:

1. On first hit for a symbol: fetch from `https://img.logokit.com/ticker/{symbol}?token=...` server-side, save the response to `static/ticker-logos/{SYMBOL}.png`, return it.
2. On subsequent hits: serve straight from the static file (no Python overhead, no upstream call).
3. Send long `Cache-Control` headers (e.g. `public, max-age=604800, immutable`) so the browser caches across page reloads.
4. Optional: 30-day TTL refresh so logos pick up branding changes.
5. On upstream failure: return a 404 (or a neutral fallback PNG) so the client renders the muted placeholder.

### Client change

One-liner: update `tickerLogoUrl()` in `frontend/src/lib/constants.js` to point at our endpoint instead of LogoKit directly. The `TickerLogo.vue` component, in-memory dedupe, and "Logos by LogoKit" attribution all stay as-is.

### Benefits

- **Cost / rate-limit safety**: cap LogoKit hits regardless of traffic
- **Performance**: same-origin, proper cache headers — second visit is free
- **Privacy**: LogoKit no longer sees end-user IPs / referrers
- **Resilience**: after warm-up, logos work even if LogoKit is down
- **CORS**: same-origin removes the class of issues that bit us during OPT-258 (LogoKit returning 403 to programmatic `fetch()`)

### Out of scope

- Replacing LogoKit as the upstream source. Attribution stays in the footer.
- Pre-warming the cache for popular tickers (can be added later if needed).
