# OPT-246: Landing page SEO: meta tags, OG, JSON-LD, favicon

Landing page (`landing/index.html`) is missing core SEO metadata. Audit found:

## Quick wins (this issue)

- [ ] Add `<meta name="description">`
- [ ] Add Open Graph tags (`og:title`, `og:description`, `og:image`, `og:url`, `og:type`, `og:site_name`)
- [ ] Add Twitter Card tags (`twitter:card`, `twitter:title`, `twitter:description`, `twitter:image`)
- [ ] Add `<link rel="canonical">`
- [ ] Add JSON-LD `SoftwareApplication` structured data
- [ ] Add favicon + apple-touch-icon links
- [ ] Add `theme-color` meta
- [ ] Trim `<title>` to \~60 chars for better SERP display

## Follow-up (separate issues)

* Precompile Tailwind instead of CDN (LCP/CWV)
* Self-host or preload Google Fonts
* Keyword-targeted H2 copy ("tastytrade trade journal", "options roll tracker")
* `robots.txt` + `sitemap.xml` served from FastAPI root
* Real product screenshots with descriptive alt text

## Why

Social link previews currently render without a description or image. Search engines auto-generate snippets from body text, which hurts CTR. Structured data helps Google render rich results.
