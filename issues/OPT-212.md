# OPT-212: Account selection for Tastytrade sync

## Summary

Allow users with multiple Tastytrade accounts to choose which accounts to sync. Currently the sync imports all accounts with no filtering.

## Current State

* `Account` model already has an `is_active` column (always set to `True`)
* `TastytradeClient` already supports per-account fetching (`get_transactions(account_number=...)`, `get_positions(account_number=...)`)
* Sync endpoints currently process all accounts unconditionally
* `GET /api/accounts` endpoint exists

## Implementation Plan

### Backend

1. Use existing `is_active` column on `Account` as the sync filter (no migration needed)
2. Filter sync endpoints (`/api/sync`, `/api/sync/initial`) to only process `is_active=True` accounts
3. Add `POST /api/settings/accounts` endpoint to toggle `is_active` per account
4. Default new accounts to `is_active=True` (backward compatible)

### Frontend

5. Add "Accounts" tab in Settings page
6. New `useSettingsAccounts.js` composable
7. Display checkboxes per discovered account (account name, type, number)
8. Require at least one account enabled before sync

### Key Files

* `src/routers/sync.py` — add account filtering
* `src/routers/accounts.py` — add preferences endpoint
* `frontend/src/pages/settings/App.vue` — add Accounts tab
* `frontend/src/pages/settings/useSettingsAccounts.js` — new composable
