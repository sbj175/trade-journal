# OPT-178: Ongoing: Security hardening and vulnerability review

## Summary

Ongoing issue to track security improvements and vulnerability fixes across the application. Keep open as a living checklist.

## Completed

* **Dotfile path blocking**: SPA catch-all route now returns 404 for `/.env`, `/.git/*`, and any path with dotfile segments (previously returned 200 with SPA HTML shell — no actual file contents were exposed, but the 200 response was misleading)

## To Review

* Audit all static file serving paths for unintended exposure
* Review CORS configuration
* Ensure all API endpoints that modify data require authentication
* Check for SQL injection in any raw query paths
* Review WebSocket authentication
* CSP headers and other security headers
* Rate limiting on auth endpoints
* Input validation on user-facing endpoints
