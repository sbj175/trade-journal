# OPT-203: Remove chain graph derivation (Stage 4) from pipeline

The chain graph (Stage 4) is no longer needed now that grouping is purely expiration-based and roll detection is done in Phase 4b.

**Approach A (this issue):** Remove Stage 4 derivation + chain cache updates. Stop populating `OrderChain`/`OrderChainCache` but leave tables in place.

* Remove `derive_chains()` call from orchestrator
* Remove `Chain` parameter from group_manager
* Remove `update_chain_cache()` calls from [sync.py](<http://sync.py>)
* Remove `source_chain_id` tracking from group_manager
* Delete `chain_graph.py`

Follow-up work: Remove `chain_service.py`, `/api/open-chains` endpoint, and refactor Positions page (Approach B).
