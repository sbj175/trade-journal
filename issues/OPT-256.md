# OPT-256: Speed on mobile

You may be correct, but one of the biggest areas where I noticed a delay is after I back out of option position detail and go back to the main screen. Just going back there's a delay.

That's almost certainly \`<KeepAlive>\` being the fix then — your Positions list component is being fully torn down and re-mounted every time you navigate back from the detail view, which means it's re-fetching and re-rendering the whole list.

The flow without KeepAlive:
> List → Detail → \*\*destroy list\*\* → back → \*\*remount list → fetch → render\*\*

With KeepAlive:
> List → Detail → \*\*pause list\*\* → back → \*\*resume list instantly\*\*

Two things to do:

\*\*1. Add KeepAlive to your router view\*\* (as shown above), or if you want to be selective and only cache specific routes:

\`\`\`vue
<RouterView v-slot="{ Component, route }">
  <KeepAlive :include="\['PositionsList'\]">
    <component :is="Component" :key="route.name" />
  </KeepAlive>
</RouterView>
\`\`\`

Make sure your Positions list component has a \`name\` option defined matching what you put in \`:include\`.

\*\*2. Use \`onActivated\` instead of \`onMounted\` for your data refresh\*\*

Once you add KeepAlive, \`onMounted\` won't fire on return. Use \`onActivated\` to do a lightweight background refresh without blocking the render:

\`\`\`javascript
onActivated(() => {
  // Refresh silently in background, don't show loading spinner
  fetchPositions({ silent: true })
})
\`\`\`

That way the user sees the cached list \*instantly\* on return, and data quietly updates behind it if anything changed.
