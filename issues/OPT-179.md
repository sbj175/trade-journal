# OPT-179: SaaS Launch: Deployment, Onboarding, CI/CD & Monetization

Continuation of the Revised Phased Plan from OPT-51. Phase 1 (SaaS-ready codebase) is complete — multi-tenancy, auth, per-user credentials, ORM, DI, Docker, test suite (218 tests). This issue covers the remaining phases needed to go live.

---

## Phase 2: Free-Tier Deployment ($0-6/mo)

- [ ] Deploy to Railway (Docker image)
- [ ] Provision Supabase PostgreSQL (free tier, 500MB)
- [ ] User onboarding flow: signup → link Tastytrade (OAuth) → first sync
- [ ] CI/CD via GitHub Actions (auto-deploy on merge to main)
- [ ] Custom domain + SSL (Railway provides free SSL)

**Exit criteria:** 5-10 active beta users providing feedback.

## Phase 3: Monetization (still $0-6/mo infra)

- [ ] Stripe integration (Checkout, webhooks, Customer Portal)
- [ ] Subscription tier feature gating (middleware)
- [ ] Frontend billing UI (plan selection, upgrade/downgrade)
- [ ] Email notifications (Resend or SES free tier) — welcome, receipts, sync alerts

**Exit criteria:** First paying customers. Track MRR.

## Phase 4: AWS Migration (when revenue justifies it)

Trigger: \~10+ paying users at $29/mo = $290/mo MRR.

- [ ] Infrastructure as Code (Terraform or CDK)
- [ ] Data migration (pg_dump → RDS)
- [ ] Move credentials to Secrets Manager
- [ ] Production monitoring (CloudWatch, Sentry)
- [ ] Security hardening (WAF, rate limiting, CORS)
- [ ] Disaster recovery (multi-AZ, backups)

## Phase 5: Growth & Hardening (ongoing)

- [ ] Admin dashboard
- [ ] Compliance (GDPR, ToS, Privacy Policy)
- [ ] Advanced monitoring & alerting
- [ ] Feature expansion based on user feedback
