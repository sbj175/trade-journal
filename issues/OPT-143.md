# OPT-143: Dockerization + Docker Compose for dev environment

Add Dockerfile, rewrite docker-compose.yml for full dev environment (app + admin + postgres + redis), create entrypoint script with auto-migrations, .dockerignore, and .env.example template.

Spawned from OPT-51 SaaS migration plan.

## Comments

### 2026-02-28 — Steve Johnson

## Implementation Summary

### Files Created

| File | Purpose |
| -- | -- |
| `Dockerfile` | Multi-stage build (python:3.12-slim), non-root `appuser`, \~392MB image |
| `docker-compose.yml` | Rewritten with 4 services: app, admin, postgres, redis |
| `docker-entrypoint.sh` | Runs Alembic migrations then starts uvicorn (parameterized via env vars) |
| `.dockerignore` | Excludes venv, .env, .git, **pycache**, .db files, etc. |
| `.env.example` | Template with all env vars documented by section |

### Files Modified

| File | Change |
| -- | -- |
| `requirements.txt` | Added `redis>=5.0` |

### Key Design Points

* **Bind mount** (`./:/app`) enables live reload without rebuilding
* `SKIP_MIGRATIONS=1` on admin service — only the app service runs migrations
* `DATABASE_URL` override in compose `environment:` uses `postgres` hostname for inter-container DNS while `.env` value (`localhost`) still works for non-Docker local dev
* **Existing workflow unchanged** — `./start.sh` with venv + SQLite continues to work

### Usage

```bash
docker compose up --build        # Start all 4 services
docker compose up postgres       # Just postgres (existing behavior)
docker compose down              # Stop services
docker compose down -v           # Stop + remove volumes
docker compose down && docker compose up # Stop and restart (like if config changes)
```

### Verification

```bash
curl http://localhost:8000/api/health          # App health check
docker compose exec redis redis-cli ping       # Redis → PONG
```
