# OPT-141: Fix Alpine error: duplicate :key in option legs x-for causes rendering failure

## Bug

Browser console shows:

```
Alpine Expression Error: can't access property "after", C is undefined
Expression: "[...(group.positions || [])].sort(..."
```

## Root Cause

The option legs `x-for` loop in `positions.html:449` uses `:key="leg.symbol"` as the unique key. When a position group has multiple lots for the same option symbol (e.g., two separate short call lots for `IBIT 260306C00039000`), Alpine's DOM reconciliation fails because the keys are duplicated.

This causes the entire expanded section to fail to render, making open option legs invisible to the user.

## Fix

Change `:key="leg.symbol"` to `:key="leg.lot_id || leg.symbol"` — `lot_id` is unique per lot (populated from `PositionLot.id` in the API response).
