# OPT-260: Strategy engine missing Call/Put Butterfly pattern (3-leg 1:2:1)

## Problem

The strategy engine has no pattern for a single-type 3-leg **Butterfly** (1× long lower wing, 2× short body, 1× long upper wing, all same type + same expiration). The `Iron Butterfly` pattern covers only the 4-leg mixed-type variant (2 calls + 2 puts with matching body strikes).

A 1:2:1 (or 2:4:2, etc.) same-type butterfly is a fundamental retail options strategy. Missing it forces the partition algorithm to break the legs into smaller matches — producing nonsensical decompositions like `Long Strangle` + `ZEBRA` on what was clearly a butterfly.

## Repro

* Account: Roth IRA ([sbj175@gmail.com](<mailto:sbj175@gmail.com>))
* Symbol: SPY, expiration 2025-07-18
* Trade: 2× butterfly, 555P long / 565P short / 570P long, opened 2025-06-02, closed 2025-06-17
* Observed: legs scattered across two groups as `Bull ZEBRA` (2×555P / 4×565P) and `Long Strangle` (2×570P + other)
* Expected: a single `Put Butterfly` group

## Where to fix

* Add `Call Butterfly` and `Put Butterfly` to `STRATEGIES` registry in `src/pipeline/strategy_engine/constants.py`
* Add a 3-leg same-type matcher in `patterns_multi.py` (or a dedicated `patterns_butterfly.py`) with rules:
  * exactly 3 legs, all options, same `option_type`, same `expiration`, strikes distinct, sorted `(low, mid, high)`
  * quantities: `low.qty == high.qty == k` and `mid.qty == 2k` for some positive integer `k`
  * wings long + body short → Long Butterfly
  * wings short + body long → Short Butterfly
* Score should match iron-butterfly tier so it wins over 1-leg + 2-leg decompositions
