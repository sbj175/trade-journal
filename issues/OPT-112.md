# OPT-112: Add a configuration banner to server logging

…to indicate server configuration at startup.

For example
Backend DB = Postgres
Supabase authentication enabled = TRUE

Plus anything else that's configurable and would be useful to know without putting sensitive info in the log.

## Comments

### 2026-02-26 — Steve Johnson

Implemented and merged to main. The banner prints at startup after `initialize_database()`:

```
====================================================
  OptionLedger v1.0.0
====================================================
  Database           : PostgreSQL
  Auth (Supabase)    : ENABLED
  OAuth2 flow        : configured
  Encryption key     : set
  Mode               : multi-user (per-user credentials)
====================================================
```

In single-user mode it shows:
```
  Auth (Supabase)    : disabled
  TT credentials     : found in .env
  Mode               : single-user
```

No sensitive values are logged — only presence/absence indicators.
