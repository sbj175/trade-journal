# OPT-139: Beta Capacity Gate

Cap signups at X users, show "beta is full" page when cap is reached, and collect waitlist emails in a local DB table.

* `BETA_MAX_USERS` env var controls cap (0 = unlimited)
* `/api/auth/config` returns `beta_open` and `beta_spots_remaining`
* Login page shows waitlist form when beta is full on sign-up tab
* Backend safety net: `ensure_user_exists()` raises 403 for new users when cap reached
* `authFetch()` catches 403 `beta_full` responses and redirects to `/beta-full`
* Waitlist emails stored in `waitlist_entries` table
