# OPT-162: Create shared design token system

## Context

The UI refresh (OPT-161) introduced a consistent "Refined Terminal" aesthetic across all pages, but design values are scattered: hex colors are hardcoded in `main.css` (\~25 instances), `tailwind.config.js` defines the palette inline, `risk/App.vue` has \~30 chart-specific hex colors, and spinner CSS is duplicated across 3 page files. Non-tv Tailwind color classes (purple-400, orange-400, yellow-400, green-400, etc.) are used in \~25 places instead of the tv-\* palette. This plan creates a single source of truth so every value flows from one file.

## Approach

**JS tokens file** ΓåÆ imported by `tailwind.config.js` (for Tailwind classes) and `risk/App.vue` (for ApexCharts configs). CSS `theme()` function replaces all hardcoded hex values in `main.css`. No CSS custom properties needed (dark-only theme, no runtime switching).

## Phase 2: Rewire `frontend/tailwind.config.js`

* Import `colors`, `shadows`, `fonts` from `design-tokens.js`
* Replace inline `tv: { ... }` object with `tv: colors`
* Replace inline `boxShadow` with `shadows`
* Replace inline `fontFamily` with `{ sans: fonts.sans, mono: fonts.mono }`
* Keep animations/keyframes/transitions inline (they're config-only, not shared)

## Phase 3: Convert `frontend/src/styles/main.css` to `theme()` calls

Replace every hardcoded hex value with `theme('colors.tv.<name>')`. Examples:

| Current | Replacement |
| -- | -- |
| `background: #0c0e14` | `background: theme('colors.tv.bg')` |
| `color: #e1e3e8` | `color: theme('colors.tv.text')` |
| `background: #161a23` | `background: theme('colors.tv.panel')` |
| `border-color: #252a36` | uses `theme('colors.tv.border')` |
| `color: #5b8def` | `color: theme('colors.tv.blue')` |
| `color: #7a8194` | `color: theme('colors.tv.muted')` |
| `color: #34d399` | `color: theme('colors.tv.green')` |
| `background: #252a36` (scrollbar) | `background: theme('colors.tv.border')` |
| `background: #2f3545` (scrollbar hover) | `background: theme('colors.tv.border-light')` |
| `background: #111318` (detail-panel) | `background: theme('colors.tv.bg-elevated')` |
| `rgba(91, 141, 239, ...)` patterns | `theme('colors.tv.blue / 0.15')` etc. |
| `rgba(52, 211, 153, ...)` patterns | `theme('colors.tv.green / 0.12')` etc. |
| `rgba(37, 42, 54, 0.4)` (row-hover) | `theme('colors.tv.border / 0.4')` |
| `rgba(22, 26, 35, 0.82)` (glass-nav) | `theme('colors.tv.panel / 0.82')` |

Note: `rgba()` patterns that use raw RGB values need conversion to Tailwind's alpha syntax: `theme('colors.tv.blue / 0.15')`.

## Phase 4: Consolidate spinner CSS

Add spinner class to `main.css` `@layer components`:

```css
.spinner {
  border: 2px solid theme('colors.tv.border');
  border-top: 2px solid theme('colors.tv.blue');
  border-radius: 50%;
  width: 28px;
  height: 28px;
  animation: spin 0.8s linear infinite;
}
@keyframes spin { 0% { transform: rotate(0deg); } 100% { transform: rotate(360deg); } }
```

Then remove the duplicated spinner `<style>` blocks from:

* `frontend/src/pages/ledger/App.vue`
* `frontend/src/pages/reports/App.vue`
* `frontend/src/pages/risk/App.vue`

Keep the ApexCharts tooltip CSS overrides in risk/App.vue (they're chart-specific).
