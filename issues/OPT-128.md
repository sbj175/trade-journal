# OPT-128: Improve Ledger display for ACAT transfers and general UI polish

**Context:** Comparing Wingman's Stock (Long) display for an MSTR position that came over via ACAT transfer vs our Ledger page.

## ACAT Transfer Display

Tastytrade's API returns ACAT transfers with `price: 341.81` (FMV at transfer date) but `value: 0.0` (no cash exchanged). Wingman shows the zero value ("what you paid"), while we show the per-share price ("what it was worth"). Both are defensible — our approach is arguably better for cost basis tracking — but it's worth considering which is more intuitive.

The action label also differs: we show `BTO` for the ACAT opening, while Wingman shows `ASSIGNED`. Since the raw transaction has `transaction_type: Receive Deliver` and `transaction_sub_type: ACAT`, we have the data to display a more accurate label like `TRANSFER` or `ACAT`.

## General Display Differences (Wingman vs OptionLedger)

| Aspect | Wingman | Our Ledger |
| -- | -- | -- |
| **Row structure** | Group header → single "Leg" summary → individual transactions indented | Group header → lot summary → individual lot rows |
| **Action labels** | `ASSIGNED` / `SELL` / `CLOSE` badges | `BTO` / `STC` / `ASSIGNMENT` badges |
| **Amounts** | Shows `CR` (credit) suffix, no prefix | Shows `$` prefix with `db`/`cr` suffix |
| **Date format** | `2025-03-26 (17:00)` with time in parens | `3/26 5:00p` compact format |
| **Quantity** | Signed: `900`, `-800`, `-100` | Signed: `+900`, `-100`, `-800` |
| **Per-share price** | Shown inline next to amount | Separate "Entry $" column |
| **Group summary** | Total credits/P&L in header row | Realized P&L in header |

Nothing in our data model prevents matching their layout more closely — all differences are purely presentational (HTML/CSS/Alpine template in `ledger.html`).
