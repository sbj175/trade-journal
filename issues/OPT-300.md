# OPT-300: Improve the onboarding flow

Parent issue for improvements to the onboarding flow — the path a brand-new user walks from sign-in to "first useful view of my data." Today the flow is mostly implicit: the Settings → Connection panel handles the Tastytrade connect, but the rest of the journey (account selection, first sync, where to land afterward) is undirected.

## Sub-items under consideration

### Dynamic default Settings tab

The Settings page currently always opens on the Connection tab. Once a user is connected, that tab is no longer the most useful landing. Proposed logic for the default:

| Condition | Default tab |
|---|---|
| `?onboarding=1` in URL | Connection (explicit onboarding flow) |
| User is **not** connected to Tastytrade | Connection (fix the blocker first) |
| Connected | General |

`?tab=` URL param continues to override everything. The connectionStatus check is async — initialize `activeTab` to `null` and render nothing in the content area until the resolve completes (a ~200ms blink is acceptable for a Settings page navigation).

### Other candidates (to flesh out later)

- A first-time setup wizard or checklist that walks Connection → Accounts → Import → "you're ready" rather than relying on the user to find each tab.
- Better post-import landing: today the import-success modal sends the user to `/positions/options`. Consider whether Ledger or a Reports overview is more rewarding as the "first real view of my data."
- Empty-state coaching on every page that depends on synced data (e.g. Ledger says "No groups" when un-synced — could point at /settings/import instead).
- Distinguishing "never connected" vs "connected but credentials expired" — they need different UX paths through Settings → Connection.
- Telemetry / opt-in event capture so we can see where new users drop off in the funnel.
