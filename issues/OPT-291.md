## Summary

The TAGS / SIGNALS column on the Options positions list rendered automated badges ("Near Short", "Elevated Delta", etc.) from the rules engine in `frontend/src/lib/rules.js`. These weren't useful in practice, so they're removed from the list view.

## Changes

- `frontend/src/components/PositionsDesktopHeader.vue` — column header renamed "Tags / Signals" → "Tags"
- `frontend/src/components/PositionsDesktopRow.vue` — dropped the badges template inside the tags cell
- `frontend/src/components/PositionsMobileCard.vue` — dropped the badges loop on the mobile card

The underlying rules engine (`lib/rules.js`, `composables/usePositionsAnalysis.js`) and the signals display inside the Detail view + Roll Analysis expanded panel are intentionally left in place. If you want those removed too, file a follow-up.
