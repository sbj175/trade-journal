# OPT-159: Sync notification shows symbols from all fetched transactions, not just new ones

The sync response's `symbols` field includes underlying symbols from ALL fetched transactions in the time window, but `new_transactions` only counts actually-new rows. This causes the notification to say e.g. "Imported 4 transactions on EWY, FDX, GLW, RIO, STRC, STX" when only 4 of those symbols had new transactions.

**Fix:** Have `save_raw_transactions` return the set of underlying symbols from actually-inserted rows, and use that in the sync response.
