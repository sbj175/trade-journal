# OPT-241: Deploy landing page to Cloudflare Pages

Deploy the landing page (`landing/index.html`) to Cloudflare Pages so it's hosted separately from the Railway app.

Steve already has a Cloudflare account (used for another domain).

## Steps

### 1\. Install Wrangler CLI (if not already installed)

```
npm install -g wrangler
```

### 2\. Login to Cloudflare

```
wrangler login
```

This opens a browser window for OAuth — approve it.

### 3\. Deploy the landing page

From the project root:

```
wrangler pages deploy landing/ --project-name optionledger
```

First run creates the project. You'll get a URL like `optionledger.pages.dev`.

### 4\. Verify it works

* Visit `https://optionledger.pages.dev` in your browser
* Test the waitlist form — it will fail at this point (expected, since `/api/waitlist` points to the same origin, not Railway)

### 5\. Point the custom domain (when ready)

* In Cloudflare dashboard → Pages → optionledger → Custom domains
* Add `optionledger.app`
* In GoDaddy DNS, add a CNAME record: `optionledger.app` → `optionledger.pages.dev`
* Cloudflare will verify and issue a free SSL cert automatically

### 6\. Wire up the waitlist API

Once the domain is live, you'll need the CF Pages Function to proxy `/api/waitlist` to Railway. That's when we create `landing/functions/api/waitlist.js` (already designed in OPT-239).

## Notes

* The form won't work on `*.pages.dev` until the proxy function is added (different origin than Railway)
* Each `wrangler pages deploy` is a new deployment — no app rebuild needed
* Can also connect to a GitHub repo for auto-deploy on push (Pages dashboard → Settings → Build)
