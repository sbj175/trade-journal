# OPT-186: ZEBRA strategy not detected by new strategy engine pipeline

## Bug

IBIT order 390131815 (6/17/25, account 5WZ26959) has legs: Buy 46x C50, Sell 23x C64 — a 2:1 ratio Bull ZEBRA. But it's labeled "Bull Call Spread".

## Root Cause

ZEBRA detection exists in the **old** `src/models/strategy_detector.py` but was never ported to the **new** strategy engine pipeline at `src/pipeline/strategy_engine/patterns_vertical.py`. The new `match_vertical()` only checks direction, not quantity ratios.

## Fix

Add ZEBRA ratio detection to `patterns_vertical.py` before the regular spread checks. If `low.quantity == 2 * high.quantity` (or vice versa), return the appropriate ZEBRA label.
