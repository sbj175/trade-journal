# OPT-101: Phase 1: Extract inline Alpine components and add Vite build step

Phase 1 prep work from OPT-100 (Frontend migration: Alpine.js → Vue 3). These improvements are valuable now and reduce the eventual migration scope.

## Tasks

1. **Extract inline components to separate** `.js` files — Move `positionsApp()`, `ledgerApp()`, etc. out of HTML and into `static/js/` files registered with `Alpine.data()`. Separates business logic from markup.
2. **Extract pure business logic into standalone JS modules** — P&L calculations, strategy detection, sort comparators, formatting helpers. These become testable and reusable in any framework.
3. **Add Vite as a build step** — Replace Tailwind CDN (\~600KB) with Vite + PostCSS purged build (\~10KB). Enables proper JS modules. Vite carries over directly when we switch to Vue.
4. **Adopt** `Alpine.store()` for shared state — Replace `localStorage` usages that act as cross-page state with proper Alpine stores. Maps directly to Pinia later.

## Pages (in order of complexity)

* Settings (simplest)
* Reports
* Risk
* Ledger
* Positions (most complex — \~1,500 lines, \~49 methods)
