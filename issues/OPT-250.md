# OPT-250: Landing: add robots.txt and sitemap.xml

`https://optionledger.app/robots.txt` and `/sitemap.xml` both 404 currently. Search engines will still crawl, but these files improve crawl efficiency and let you explicitly tell Google what to index.

## Plan

Create two files in `landing/`:

`landing/robots.txt`

```
User-agent: *
Allow: /

Sitemap: https://optionledger.app/sitemap.xml
```

`landing/sitemap.xml`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://optionledger.app/</loc>
    <lastmod>2026-04-15</lastmod>
    <changefreq>weekly</changefreq>
    <priority>1.0</priority>
  </url>
  <url>
    <loc>https://optionledger.app/privacy</loc>
    <changefreq>monthly</changefreq>
    <priority>0.3</priority>
  </url>
</urlset>
```

Note: Cloudflare Pages serves files from `landing/` at the root automatically, so these should Just Work.

## Verify after deploy

* `curl https://optionledger.app/robots.txt` returns the text
* `curl https://optionledger.app/sitemap.xml` returns the XML
* Submit sitemap in Google Search Console once verified
