# OPT-211: Roll chain summary table + surface on Options Positions page

## Summary

Add a `roll_chain_summary` table that materializes roll chain statistics, and surface that data on the Options Positions page. Currently, roll chain data is only available via the Ledger page's RollChainModal and requires on-demand traversal (4 queries per chain).

## Motivation

* Options Positions page has no visibility into roll history
* Computing roll chain stats on the fly for every open position would be too expensive
* Traders want to see roll count, cumulative premium, and cumulative P&L alongside their live positions

## Proposed Schema: `roll_chain_summary`

| Column | Type | Description |
| -- | -- | -- |
| `id` | Integer PK | Auto-increment |
| `user_id` | String FK | Tenant scoping |
| `root_group_id` | String | First group in chain |
| `current_group_id` | String | Latest (open) group |
| `underlying` | String | For joining to positions |
| `account_number` | String | For joining to positions |
| `chain_length` | Integer | Total groups in chain |
| `roll_count` | Integer | chain_length - 1 |
| `first_opened` | DateTime | Root group's opening_date |
| `last_rolled` | DateTime | Most recent roll date |
| `cumulative_premium` | Float | Sum of entry prices across all lots in chain |
| `cumulative_realized_pnl` | Float | Sum of realized P&L across all closings in chain |

## Implementation Plan

### Backend

1. Add `RollChainSummary` model to `src/database/models.py`
2. Create Alembic migration
3. Add pipeline stage after group processing in `orchestrator.py` to populate/update the table
4. Add API endpoint or extend `/api/positions` response to include roll chain summary data

### Frontend

5. Options Positions page: join roll chain summary data to position rows
6. Show roll indicator (e.g., "3x" badge) on positions with roll history
7. Display cumulative premium and cumulative P&L columns/tooltips
8. Wire up click handler to open existing `RollChainModal` (already a shared component)

### Reuse

* `RollChainModal` component — no changes needed, already accepts `groupId` + `underlying` props
* `/api/ledger/group-roll-chain/{group_id}` endpoint — no changes needed for modal detail view
