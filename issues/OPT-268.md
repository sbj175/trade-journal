# OPT-268: Mirror OPT-263 roll timeline on Positions page + drop order-artifact fields

## Why

[OPT-263](https://linear.app/optionedge/issue/OPT-263/rolled-position-groups-display-is-confusing-strategy-label-rolls) introduced staged Opening/Roll/Closing sections on the Ledger so rolled positions read cleanly. The Positions page shows the same underlying groups but still renders the old flat leg view, and its collapsed header still shows the historical strike union (rolled IC displays 6 strikes instead of 4). Two broker-artifact fields also leak into the Positions UI from the pre-[OPT-263](https://linear.app/optionedge/issue/OPT-263/rolled-position-groups-display-is-confusing-strategy-label-rolls) era.

This issue ports the [OPT-263](https://linear.app/optionedge/issue/OPT-263/rolled-position-groups-display-is-confusing-strategy-label-rolls) improvements to Positions, with a hybrid that preserves what the Positions page does uniquely well (live quotes, real-time P&L per open leg).

## Design

**Preserve**: the existing open-leg block with live quotes and real-time P&L at the top of the expanded panel. That is Positions' unique value over Ledger.

**Add below it** (only when `group.roll_count > 0`):

* ROLL section(s) — same styling and interaction as Ledger: descending chronology, clickable row, `ROLLED X → Y` summary, expandable per-leg pairs
* OPENING section — same styling as Ledger: date + net credit/debit, leg rows sorted by strike ascending

**No CLOSING section** — Positions lists currently-open groups by definition.

**Collapsed header**:

* Use `current_strike_label` instead of the client-side historical strike union
* Add the same roll count badge as Ledger
* Drop the `partially_rolled` ⓘ icon (same reasoning as [OPT-267](https://linear.app/optionedge/issue/OPT-267/remove-the-partially-rolled-indicator-redundant-after-opt-263): staging is now explicit, and the order_id-based heuristic is fragile)

**Also drop**: the "Orders: N" chip in `PositionsExpandedPanel.vue`. It's a broker-artifact count (how many `order_id`s the execution bundled into), not a meaningful trading concept.

## Component strategy

Rename `frontend/src/components/LedgerRollTimeline.vue` → `RollTimeline.vue` (location neutral) and add a `show` prop:

```js
defineProps({
  timeline: Object,
  show: { type: Array, default: () => ['closing', 'roll', 'opening'] },
})
```

Ledger passes the default (all three). Positions passes `['roll', 'opening']`.

## Files to change

### Backend

* `src/routers/positions.py` — in `/api/open-chains`: call `compute_roll_timeline(lots_data)` and attach `roll_timeline`, `current_strike_label`, `roll_count`; remove `order_count` (line 323) and `partially_rolled` (lines 305-310, 325); trim the now-stale comment at line 132

### Frontend

* `frontend/src/components/LedgerRollTimeline.vue` → `frontend/src/components/RollTimeline.vue` (git mv + add `show` prop)
* `frontend/src/components/LedgerDesktopRow.vue` + `LedgerMobileCard.vue` — update import path (no behavior change)
* `frontend/src/components/PositionsDesktopRow.vue` — swap `group.strikes` → `current_strike_label`, add roll count badge, remove `partially_rolled` ⓘ
* `frontend/src/components/PositionsMobileCard.vue` — same header updates
* `frontend/src/components/PositionsExpandedPanel.vue` — render `<RollTimeline v-if="group.roll_count > 0" :timeline="group.roll_timeline" :show="['roll','opening']" />` below the existing open-leg block; remove the "Orders: N" chip at line 102

## Acceptance

* Rolled IC on Positions shows 4 strikes in the header, not 6.
* Collapsed row has a roll count badge matching what Ledger shows for the same group.
* Expanded panel retains the live-quote open-leg block unchanged; below it, a Roll + Opening section appears when the group has been rolled.
* `/api/open-chains` response no longer includes `partially_rolled` or `order_count`.
* Ledger view unaffected by the component rename.

## Non-goals

* No changes to live quote streaming, WebSocket logic, or the Positions greeks display.
* Not revisiting the RollChainModal (different-exp chains are separate, handled via the `has_roll_chain` link icon which stays).
