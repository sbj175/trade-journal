# OPT-242: Add Google OAuth login via Supabase

Add "Sign in with Google" as an additional login option alongside existing email/password auth. Reduces signup friction.

## Implementation

### 1\. Google Cloud Console setup (manual)

* Create a new OAuth 2.0 credential (Web application)
* Authorized redirect URI: `https://<supabase-project>.supabase.co/auth/v1/callback`
* Save the Client ID and Client Secret

### 2\. Supabase Dashboard setup (manual)

* Authentication → Providers → Google
* Enable and paste Client ID + Client Secret from step 1

### 3\. Frontend changes

* Add "Sign in with Google" button to `static/login.html`
* Button calls `supabase.auth.signInWithOAuth({ provider: 'google' })`
* Style consistent with existing login page design

### 4\. Verify existing backend handles it

* JWT validation (`jwt_validator.py`) — should work as-is since Supabase issues the JWT regardless of provider
* User provisioning (`user_provisioning.py`) — auto-creates User row on first login, should work for Google users
* `get_current_user_id()` — unchanged

## Notes

* Google login is **in addition to** existing email/password auth
* Waitlist/beta gate still applies — Google signups should respect the beta capacity check
* Tastytrade connection is separate — users still connect Tastytrade after logging in
* No backend code changes expected — Supabase handles the Google OAuth flow entirely
