# OPT-269: Mobile header should surface the current page name

## Why

On mobile, the top nav collapses into a drawer/burger menu. When the drawer is closed (the default state) there's no indication of which page is currently showing — a user landing on `/reports` from a deep link or after tapping a link can't tell at a glance whether they're looking at the Ledger, Reports, Positions, Risk, or Settings view. Desktop has persistent nav tabs where the active link is highlighted; mobile loses that affordance entirely.

## Scope

Add a clear current-page indicator to the mobile header. Candidates to evaluate:

* Page title text in the mobile top bar (next to or replacing the logo), derived from the active route's `meta.title` or a small route-to-label map
* Active-state styling on the menu icon, or a small breadcrumb-style chip
* Route-driven page heading rendered by the page itself (in the `DefaultLayout` above the content) so every page gets a consistent top-of-content title

## Non-goals

* No redesign of the drawer itself
* No changes to desktop navigation
* No router/auth guard changes

## Acceptance

* Any page loaded on a narrow viewport shows the current page name without opening the drawer
* Switching pages updates the indicator
* The indicator uses existing typography/color tokens (no new design system introductions)

## Context

Related files:

* `frontend/src/components/NavBar.vue` — top-level nav, has `route.path`-based active logic for desktop
* `frontend/src/layouts/DefaultLayout.vue` — wraps page content; potential home for a route-driven heading
* `frontend/src/router/index.js` — route definitions; add `meta.title` to each if we go with a meta-driven title
