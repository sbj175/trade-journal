# OPT-108: Review use of browser local storage

The following items need to be reviewed to determine whether the backend DB is a better storage location.

● Here's everything stored in localStorage, grouped by purpose:

Cross-page shared state:

* trade_journal_selected_account — selected account, synced across Positions, Ledger, Reports, and Risk pages                                      - trade_journal_selected_underlying — selected underlying filter, synced across Positions and Ledger

Settings (from Settings page):

* privacyMode — privacy mode toggle ('off' / 'on'), read by Positions page too
* rollAlertSettings — roll alert DTE thresholds (JSON), read by Positions page

Positions page:

* positions_sort — sort column + direction (JSON)
* positionComments — legacy; migrated to database on first load, then removed

Ledger page:

* ledger_state — view mode, sort, and grouping preferences (JSON)

Reports page:

* reports_time_period — selected time period filter
* reports_category_filters — strategy/status filter selections (JSON)
* reports_sort — sort column + direction (JSON)

Risk page:

* Uses trade_journal_selected_account (shared)

Utility:

* clear_cache.html — a cache-clearing page that wipes all localStorage

None of this data lives in the database, so all of it persists unchanged when you switch backends. The one exception is position comments/notes,

which were migrated from localStorage into the database — those will differ between SQLite and PostgreSQL unless you run the migration script.
