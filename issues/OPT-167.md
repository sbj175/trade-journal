# OPT-167: Replace Time dropdown with date range filters on Reports page

## Summary

Replace the current Time dropdown (Today, Yesterday, 7D, 30D, 60D, 90D, All) on the Reports page with explicit date range pickers for **Entry Date** and **Exit Date**. This enables proper monthly/periodic reporting by letting users specify exactly which trades to include based on when they were opened and/or closed.

## Current Behavior

* The Reports page has a `Time` dropdown that filters closed chains by `closing_date >= (now - N days)`
* This is a rolling window — it can't target a specific month or date range
* Only filters by closing date, no way to filter by entry date

## Proposed Changes

### Frontend (`frontend/src/pages/reports/App.vue`)

1. **Remove** the Time dropdown (`<select v-model="timePeriod">`)
2. **Add two date range pickers** in the filter bar:
   * **Entry Date Range**: filters on `OrderChain.opening_date` (when trades were opened)
   * **Exit Date Range**: filters on `OrderChain.closing_date` (when trades were closed)
3. Each picker has a **From** and **To** date input (`<input type="date">`)
4. Add **quick-select presets** for common ranges (e.g., "This Month", "Last Month", "This Year", "YTD", "Last 90 Days") — these populate the date inputs rather than being a separate mechanism
5. Persist selected date ranges to `localStorage` (replacing `reports_time_period`)
6. Update `fetchReport()` to send `entry_from`, `entry_to`, `exit_from`, `exit_to` query params instead of `days`

### Backend (`src/routers/reports.py`)

1. **Update** `/api/reports/performance` endpoint:
   * Add optional query params: `entry_from`, `entry_to`, `exit_from`, `exit_to` (ISO date strings)
   * Remove the `days` parameter (or keep for backward compat initially)
   * Apply filters: `OrderChain.opening_date >= entry_from`, `OrderChain.opening_date <= entry_to`, etc.
2. Both date ranges are independently optional — user can filter by entry only, exit only, or both

### UX Notes

* Default state on first load: Exit Date = current month (1st to today) — equivalent to "this month's closings"
* Clearing both date ranges = "All" (no date filter)
* Quick presets apply to the **Exit Date** range by default (most common use case: "show me what I closed last month")
* Entry Date range is collapsed/hidden by default, expandable — keeps the UI clean for the common case

## Key Files

| File | Change |

| `src/routers/reports.py` | Update `/api/reports/performance` to accept date range params |

## Implementation Steps

1. Update backend endpoint to accept new date range params (keep `days` working for now)
2. Build date range picker UI components in the filter bar
3. Add quick-select presets
4. Wire up frontend to send date range params
5. Remove old `days` param and Time dropdown
6. Update localStorage persistence
7. Test with various date ranges and edge cases (empty ranges, future dates, etc.)
