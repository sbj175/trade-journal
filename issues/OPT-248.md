# OPT-248: Landing: self-host or preload Google Fonts

`landing/index.html` fetches three font families from `fonts.googleapis.com` (Newsreader, Outfit, IBM Plex Mono). This adds an external DNS lookup + request on every page load and triggers a FOUT/FOIT flash.

## Plan

Option A (recommended — self-host):

* Download WOFF2 files for the exact weights used (Newsreader 400/500/600/700 + italic; Outfit 300/400/500/600/700; IBM Plex Mono 400/500)
* Place in `landing/fonts/`
* Add `@font-face` rules in the precompiled CSS with `font-display: swap`
* Remove the Google Fonts `<link>` tags

Option B (minimum effort):

* Add `<link rel="preload" as="font" type="font/woff2" crossorigin>` hints for the hero-visible fonts (Newsreader bold + Outfit regular)
* Keep the existing Google Fonts CSS link

## Why

* Removes 1 extra DNS lookup + TCP/TLS handshake to `fonts.gstatic.com`
* Eliminates GDPR risk (Google Fonts hot-linking has been ruled non-compliant in EU courts)
* Improves LCP by \~100-300ms in most cases
* Privacy: Google Fonts CDN logs visitor IPs
