# OPT-295: Analysis panel — surface chain break-even close price on covered calls

When closing a covered-call roll chain leg, the trader currently has to eyeball "what BTC price keeps the chain net-flat?" The Roll Analysis panel already integrates Chain-vs-Open context (chain mode adds prior roll P&L into reward/risk), but it stops short of telling you the *price you need to see on the broker screen* to end the chain in profit.

## Scope

1. **Rename** the section header in `frontend/src/components/PositionsRollAnalysis.vue:13` from "Roll Analysis" → "Analysis". Label-only — keep the file name, prop names (`rollAnalysisMode`), and event names (`toggle-roll-analysis-mode`) untouched.

2. **Add a "Chain B/E Close" row** to the P&L Status column (first column of the existing 3-column layout). Shows the per-contract debit price at which closing the current open leg leaves the chain net-flat.

   **Calculation:**
   ```
   chain_breakeven_btc_per_contract =
     (current_leg_premium_received + cumulative_prior_chain_realized)
       / (contracts × 100)
   ```
   - `current_leg_premium_received` — gross premium dollars credited on the still-open leg (already on the group as `current_premium_received` or computed from lots)
   - `cumulative_prior_chain_realized` — sum of realized $ across all closed legs in the chain (already on `group.roll_chain.cumulative_realized_pnl`)
   - `contracts × 100` — to express the result as a per-contract debit price the user types into Tastytrade

   **Display:** only meaningful in **Chain** mode (with Open mode it equals the current leg's premium received, which is redundant). In Chain mode show the price; in Open mode either hide the row or grey it out with the same caveat tooltip.

3. **Tooltip copy:**
   > Buy-to-close at or below this price keeps the entire chain (this leg + all prior rolls) net-flat or in profit.

## Out of scope

- Live BTC quote delta ("$0.84 below mid") — leave for a follow-up once the static number is right.
- Chain break-even rendering inside `RollChainModal.vue`.
- Equity-side or naked-call variants — covered-call (short call) only for now.

- Cross-check against the Roll Chain modal: `chain_breakeven_btc * contracts * 100 = current_premium_received + cumulative_realized_prior`.
