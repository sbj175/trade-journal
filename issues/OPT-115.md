# OPT-115: Preserve user group assignments across reprocess-chains

When reprocess-chains runs, `clear_all_lots()` now also clears `PositionGroup` and `PositionGroupLot` tables (added in OPT-114 fix). This means any manual lot moves made by users on the Ledger page are lost after reprocessing.

This is acceptable for now since reprocess-chains is an admin-only function not exposed to users. Normal syncs do incremental updates and don't clear groups.

**If this becomes needed**, consider:

1. Snapshot user-modified group assignments before clearing, replay after re-seeding
2. Add a `user_modified` flag to skip clearing user-customized groups
3. Separate overrides table that survives clearing

Low priority — only relevant if reprocess becomes an admin tool or if incremental sync ever needs a full rebuild path.
