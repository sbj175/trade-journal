# OPT-239: Prelaunch wait-list

Yes—**a landing page on CloudFlare Pages can absolutely submit waitlist emails to your backend running on railway.** That's a very common setup.

---

## Use Cloudflare Pages Functions as a proxy (recommended architecture)

Cleaner and more secure long-term.

Flow:

```
Visitor
→ Cloudflare Pages Function
→ Railway backend
```

Advantages:

* hides your Railway endpoint
* lets you validate emails server-side
* prevents spam bots hitting Railway directly
* supports rate limiting later

Example structure:

```
/functions/waitlist.js
```

Example function:

```
export async function onRequestPost(context) {
  const data = await context.request.json();

  const response = await fetch("https://api.optionledger.app/waitlist", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(data)
  });

  return new Response("OK");
}
```

Then your form submits to:

```
/waitlist
```

instead of Railway directly.

This is usually the best setup for a SaaS-style landing page like **optionledger.app**.
