# OPT-247: Landing: precompile Tailwind instead of CDN

`landing/index.html` currently loads Tailwind via `https://cdn.tailwindcss.com`, which hurts Core Web Vitals (LCP, CLS) and ships thousands of unused utility classes.

## Plan

* Add a minimal Tailwind build for the landing page (postcss + tailwind CLI, or inline `@tailwindcss/cli`)
* Output a single purged CSS file into `landing/` (e.g., `styles.css`)
* Replace the `<script src="cdn.tailwindcss.com">` tag with `<link rel="stylesheet" href="styles.css">`
* Move the inline `tailwind.config` theme extension into `tailwind.config.js`
* Verify Cloudflare Pages build works (may need a `package.json` + build command in the Pages dashboard, or precompile locally and commit)

## Why

Cloudflare Pages hosting + CDN Tailwind means the browser downloads \~3MB of JS on every page load just to generate CSS at runtime. A precompiled file will be \~10-20KB. Directly improves LCP, which Google uses as a ranking signal.
