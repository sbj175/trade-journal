# OPT-201: Lot-level Roll Chain Tracking (Phase 1 & 2)

## Context

Roll chains are currently invisible in the data model. When a ROLLING order is processed, `close_lot_fifo()` creates `LotClosing` records and `create_lot()` creates new `PositionLot` records, but nothing links them. The `resulting_lot_id` column on `LotClosing` exists but is only populated for assignments/exercises. Populating it for rolls enables roll chain traversal.

## Phase 1: Backend Plumbing

* Capture `create_lot()` return values for ROLLING orders in `process_lots()`
* Collect closing transaction IDs for ROLLING orders
* Add `_link_roll_resulting_lots()` helper to match and link closings to new lots

## Phase 2: API Endpoint

* Add `get_roll_chain()` method to LotManager (walk backward/forward through resulting_lot_id links)
* Add `GET /api/ledger/roll-chain/{lot_id}` endpoint

No schema changes needed — `resulting_lot_id` column already exists.
