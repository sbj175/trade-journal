# OPT-163: Convert MPA to Vue Router SPA

## Summary

Converted the multi-page app (MPA) with 7 separate Vite entry points to a single-page app (SPA) using Vue Router 4. This eliminates the white flash on page navigation, deduplicates the nav bar (\~100 lines x 7 pages), and centralizes auth checks in a router guard.

## Changes

### New Infrastructure

* `frontend/src/main.js` — single SPA entry point
* `frontend/src/App.vue` — root component (`<router-view />`)
* `frontend/src/router/index.js` — routes with lazy-loaded pages + auth guards
* `frontend/src/layouts/DefaultLayout.vue` — NavBar + `<router-view />`
* `frontend/src/components/NavBar.vue` — shared nav bar using `<router-link>`
* `frontend/src/components/AccountSelect.vue` — shared account selector
* `frontend/src/stores/auth.js` — Pinia store for auth state
* `frontend/src/stores/accounts.js` — Pinia store for accounts + selection
* `static/index.html` — single HTML shell

### Migrated Pages (all 7)

* Removed nav bar template (\~100 lines each)
* Removed `navLinks`, `authEnabled`, `userEmail` refs
* Removed `Auth.requireAuth()` / `Auth.requireTastytrade()` from `onMounted()`
* Replaced `window.location.search` with `useRoute().query` (Ledger, Settings)
* Replaced `window.location.href = '/positions'` with `router.push()` (Settings)
* Risk page: added dynamic ApexCharts loading via `ensureApexCharts()`

### Backend

* `src/routers/pages.py` — catch-all `/{full_path:path}` serving `index.html`
* `app.py` — moved `pages.router` to last position (after all API routers)

### Deleted

* 7 old HTML shells (`positions.html`, `ledger.html`, etc.)
* 7 per-page `main.js` entry points

### Vite Config

* Single `app` entry replaces 7 entries; Vite auto code-splits per page
