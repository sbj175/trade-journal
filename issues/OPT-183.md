# OPT-183: Research: Add EV (Expected Value) to Roll Analysis

## Research: EV in Roll Analysis

### Summary

All building blocks exist in the codebase. Implementation is \~15-20 lines in `getRollAnalysis()`.

### Approach

Use `abs(shortLeg.delta)` as probability of the short leg finishing ITM (standard practice, implicitly accounts for IV skew):

```
P_loss = abs(shortDelta)
P_profit = 1 - P_loss
EV = (P_profit × maxProfit) + (P_loss × maxLoss)
```

Display as dollar value + optional "EV/Risk" percentage in the Greeks or Context section.

### What it adds

Forward-looking probabilistic signal that complements current P&L-based and Greeks-based indicators:

* **Positive EV + high profit captured** → strong close signal
* **Negative EV + loss incurred** → urgent roll/close signal
* **Positive EV + loss incurred** → hold signal (odds still favor you)

### Considerations

* Binary outcome model (max profit or max loss) is a simplification but industry-standard
* Using broker delta for ITM probability is more accurate than raw BS d2 (accounts for skew)
* Transaction costs not modeled (could optionally subtract)
* EV is to-expiration; doesn't account for early management targets
