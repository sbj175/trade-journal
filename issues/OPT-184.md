# OPT-184: Rules-based advice engine for Roll Analysis

## Overview

Replace/augment the current simple suggestion logic in Roll Analysis with a structured rules-based engine. Options position management has well-defined heuristics — a deterministic rules engine could cover \~80% of meaningful advice without API costs.

## Proposed Rules

| Condition | Action |
| -- | -- |
| Profit % ≥ 50% | Suggest close |
| Near Short < 5% AND DTE > 21 | Suggest roll or close |
| Reward:Risk < 1:1 AND loss incurred > 20% | Suggest roll or close evaluation |
| DTE < 21 AND in loss | Escalate urgency |
| Delta Sat > 40% | Flag directional risk |
| Theta positive AND DTE > 45 AND Near Short > 15% | Hold signal |

## Benefits

* **Consistent, explainable advice** — users can understand why the system recommends an action
* **No API costs** — runs entirely client-side on every page load
* **Fast** — deterministic evaluation, no latency
* **Extensible** — easy to add/tune rules over time

## Future Consideration

AI could be layered on top for edge cases or conversational explanations (the existing "AI Insight" button placeholder), but the core insight engine would be entirely deterministic.

## Current State

`getRollAnalysis()` already has basic suggestion logic (profit target hit, loss limit hit) and badges (late stage, low R:R). This would formalize and expand that into a proper multi-rule evaluation with prioritized, composable signals.

Example:

| Rule | Values | Fires? |
| -- | -- | -- |
| Profit % ≥ 50% | \-25% loss | No |
| Near Short < 5% AND DTE > 21 | 0.3% AND 39d | **Yes → Suggest roll or close** |
| Reward:Risk < 1:1 AND loss > 20% | 0.8:1 AND 25% | **Yes → Suggest roll or close evaluation** |
| DTE < 21 AND in loss | 39 DTE | No |
| Delta Sat > 40% | 43.2% | **Yes → Flag directional risk** |
| Theta positive AND DTE > 45 AND Near Short > 15% | DTE only 39 | No |

Three rules fire, none contradict each other — they all point the same direction. That's a clean signal.

The output would be something like: **"Directional risk elevated. Short strike at-the-money with unfavorable reward/risk and 25% loss incurred. Consider rolling or closing."**

---

What it actually means in plain English is: **"The short strike has a 43% chance of finishing in the money."**

That's what Delta Sat is telling you — the short leg's delta is essentially the market's probability estimate of that strike being breached at expiration. Saying "directional risk elevated" obscures something that's actually very concrete and actionable.

Better phrasing for the rule output: **"Short strike has a 43% probability of finishing ITM."**
