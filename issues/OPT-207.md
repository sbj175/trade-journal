# OPT-207: Ledger: Close Status detail + group by close disposition

## Context

When viewing closed positions on the Ledger, we currently show a single "CLOSED" status badge and a blended average close price per strike. This hides important detail when lots at the same strike closed in different ways (e.g., some sold to close, some exercised, some expired).

**Example — IBIT Bull ZEBRA (Dec 31, 2025 expiration):**

* 8x Long $47 Call @ $18.37 entry
  * 2 sold to close on 7/25/2025 @ $21.93
  * 6 exercised on 12/31/2025 (no close price — shares delivered)
* 4x Short $61 Call @ $9.94 entry
  * 4 bought to close on 11/24/2025 @ $3.32

Currently the Ledger shows:

```
8   Dec 31   47   Call   CLOSED   $18.37   $5.48
```

The $5.48 close price is a misleading weighted average of the $21.93 sale and $0 exercise.

## Inspiration

OptionIncome handles this with a **Close Status** column that summarizes the disposition:

> Closed (2, 7/25/2025), Exercised (6, 12/31/2025)

They also split legs with different close types into separate rows, so entry/close prices are always meaningful.

## Proposed Approach

### 1\. Group by (strike + close disposition), not just strike

When lots at the same strike have different closing types (MANUAL, EXERCISE, EXPIRATION, ASSIGNMENT), split them into separate display rows:

```
Qty   Exp     Strike  Type  Status    Entry    Close    Close Status
2     Dec 31  47      Call  CLOSED    $18.37   $21.93   Closed (7/25/2025)
6     Dec 31  47      Call  CLOSED    $18.37   —        Exercised (12/31/2025)
-4    Dec 31  61      Call  CLOSED    $9.94    $3.32    Closed (11/24/2025)
```

* If all lots at a strike closed the same way → one row (no change from current)
* If lots closed differently → split into sub-rows, one per disposition
* Exercise/assignment rows show `—` for close price (no market close)

### 2\. Add Close Status column

A text column on closed lot rows that summarizes the closing events. Format: `{Type} ({qty}, {date})`, comma-separated if multiple close dates for the same disposition.

### 3\. Data source

All closing detail is already in the `lot_closings` table:

* `closing_type`: MANUAL, EXERCISE, EXPIRATION, ASSIGNMENT
* `quantity_closed`: how many contracts
* `closing_date`: when
* `closing_price`: at what price
* `resulting_lot_id`: links to derived equity lot (for exercises)

The `/api/ledger` endpoint needs to include closing event summaries in its response. The frontend grouping logic (`groupedOptionLegs` or equivalent) needs to group by `(expiration, strike, option_type, closing_type)` instead of just `(expiration, strike, option_type)`.

| File | Change |
| -- | -- |
| `src/routers/ledger.py` | Include lot closing details in API response |
| `frontend/src/pages/ledger/App.vue` | Group by disposition, add Close Status column, show `—` for exercise/assignment close price |
