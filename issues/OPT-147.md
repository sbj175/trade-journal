# OPT-147: Fix account balances multi-tenant scoping bug

**Bug**: Account balances appear shared across users — one user's sync overwrites another user's balances.

**Root cause**: `get_account_balances()` in `src/routers/accounts.py` uses a subquery to find the latest timestamp per account, but this subquery is NOT scoped by `user_id`. The ORM tenant filter (`do_orm_execute` listener) only applies to the outer query, not subqueries. This means the subquery's `MAX(timestamp)` considers ALL users' balance rows, causing the JOIN to fail for users whose balance doesn't have the globally-latest timestamp for that account number.

Additionally, `save_account_balance()` in `src/database/db_manager.py` doesn't explicitly set `user_id` (relies on ORM `before_flush` listener, which is fragile for this pattern).

**Fix**:

1. Add explicit `user_id` filter to the subquery in `get_account_balances()`
2. Add explicit `user_id` to `save_account_balance()` in db_manager
