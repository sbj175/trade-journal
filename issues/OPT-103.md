# OPT-103: Newly added shares should be added into the existing group where shares are already (assuming that exists).

For example, I had IBIT shares in an existing Covered Call group, bought more shares, and those shares were put into a new group.

## Comments

### 2026-02-23 — Steve Johnson

Fixed and merged to `main` in commit `d2b55f4`.

**Root cause:** The queries in `seed_new_lots_into_groups()` and `seed_position_groups()` only looked for groups with `source_chain_id IS NULL AND strategy_label = 'Shares'`, missing Covered Call groups (which have a `source_chain_id` and `strategy_label = 'Covered Call'`).

**Fix:** Both queries now find any OPEN group for the same account+underlying:
```sql
SELECT group_id FROM position_groups
WHERE account_number = ? AND underlying = ? AND status = 'OPEN'
ORDER BY opening_date DESC LIMIT 1
```

A new "Shares" group is only created if no OPEN group exists at all.

**Tests:** Added 4 new tests in `tests/unit/test_ledger_service.py` covering:
- New equity lot joins existing Covered Call group
- New equity lot creates Shares group when no OPEN group exists
- New equity lot ignores CLOSED groups
- Initial seeding merges chainless lots into existing OPEN group

All 71 tests pass.
