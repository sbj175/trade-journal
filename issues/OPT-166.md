# OPT-166: Replace quote polling with true TT DXLink streaming

**Current behavior:** The WebSocket endpoint (`/ws/quotes`) polls Tastytrade via `client.get_quotes()` every 5 seconds and pushes results to the frontend. This is not true streaming.

**Desired behavior:** Open a persistent `DXLinkStreamer` connection to Tastytrade and forward quote/trade/greeks events directly to the frontend WebSocket as they arrive — real-time, sub-second updates with no polling.

**Key files:**

* `src/routers/quotes.py` — WebSocket endpoint with 5-second polling loop (`send_updates()`)
* `src/api/tastytrade_client.py` — `_async_fetch_quotes()` already uses `DXLinkStreamer` for one-shot fetches; this pattern could be adapted for persistent streaming

**Considerations:**

* Connection lifecycle management (reconnect on drop, clean shutdown)
* Per-user streamer instances when auth is enabled
* Throttling/batching updates to avoid overwhelming the frontend
* Graceful fallback if DXLink connection fails
