# OPT-140: Fix: Positions page 500 error — string vs date comparison on lot expiration

The `/api/open-chains` endpoint compares `lot.expiration` (a `String` column) directly against `date.today()`, which raises `TypeError: '<' not supported between instances of 'str' and 'datetime.date'` in Python 3. This causes the entire endpoint to return 500, hiding all positions on the Positions page.\\n\\nRoot cause: `PositionLot.expiration` is `Column(String)` but the filter at `positions.py:213` does `lot.expiration < date.today()` without parsing.\\n\\nReproduced with IBIT short calls (strikes 40/41/42, exp 2026-02-27) for user [sbj175@gmail.com](<mailto:sbj175@gmail.com>), account 5WZ28644.
