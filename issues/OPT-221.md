### 2026-03-30 — Steve Johnson

## Root Cause Analysis

The database `quote_cache` table had **no TTL enforcement**. `get_cached_quotes()` returned any cached row regardless of age.

### How staleness persisted:
1. Container restart clears in-memory cache (30s TTL)
2. Frontend requests `/api/quotes` without `refresh=true`
3. Endpoint returns DB-cached quotes — could be **days old** — as if they were fresh
4. WebSocket `send_updates()` loop writes to DB every 5 seconds, but only for symbols that successfully fetch from the API
5. Symbols that fail to fetch keep their old DB cache entry indefinitely
6. Only fix was manually deleting `quote_cache` rows

### Fix
Added `max_age_seconds` parameter (default 5 minutes) to `get_cached_quotes()`. The method now filters `WHERE updated_at >= cutoff`. All 4 call sites (quotes REST, quotes WebSocket fallback, positions cached, ledger) automatically benefit.

After TTL expiry, stale quotes are ignored and the endpoint either fetches fresh data or returns an appropriate error — no more silently serving week-old prices.

### 2026-03-30 — Steve Johnson

## Additional Fixes — Mobile Spinners

The DB cache TTL fix (commit 1) addressed stale prices persisting after restart. But the **persistent spinners on mobile** had three additional root causes:

### 1. Equities WebSocket was completely broken (critical)
`useEquityQuotes.js` sent `{ type: 'subscribe', symbols }` but the backend expects `{ subscribe: symbols }`. The subscription was silently ignored. Also parsed `msg.type === 'quote'` (singular) but backend sends `'quotes'` (plural) with nested `msg.data`. **Live quotes literally never worked on the equities page via WebSocket.**

### 2. Equities had zero reconnect logic
`ws.onclose` just set `liveQuotesActive = false` and gave up. One disconnect = permanent loss of live quotes.

### 3. No visibility change handling on either page
Mobile browsers kill WebSocket connections when the tab is backgrounded. Neither page detected `visibilitychange` to reconnect when the user returns. Combined with the 5-attempt reconnect limit on options page (and zero attempts on equities), mobile users would permanently lose live quotes.

### Fix (commit 2)
- Fixed equities WebSocket message format and response parsing
- Added reconnect logic to equities (5 attempts with exponential backoff, matching options page)
- Added `visibilitychange` listener to both pages — reconnects WebSocket when tab becomes visible
- Both pages now fall back to REST polling every 15s after WebSocket retries exhaust
