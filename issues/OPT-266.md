# OPT-266: Explain the roll model in-UI — help users understand same-exp vs different-exp rolls

## Why

OptionLedger intentionally represents rolls two different ways depending on whether the expiration changes (see `project_roll_model_decision.md`, [OPT-263](https://linear.app/optionedge/issue/OPT-263/rolled-position-groups-display-is-confusing-strategy-label-rolls)):

* **Same-expiration roll** (strike adjustment) → stays inside one `PositionGroup`. Treated as **position management** of a continuing trade. Rendered as staged sections within one card ([OPT-263](https://linear.app/optionedge/issue/OPT-263/rolled-position-groups-display-is-confusing-strategy-label-rolls)).
* **Different-expiration roll** (time extension) → creates a new `PositionGroup` linked via `rolled_from_group_id`. Treated as a **new trade**. Rendered as a chain of cards; details live in `RollChainModal`.

Both shapes show up side-by-side in the Ledger, and the two UIs look quite different. Without in-app explanation, a user can reasonably wonder:

* "Why did rolling my 530/545 put credit spread to 585/600 keep it in one card, but rolling the same spread from Jul → Aug split it into two?"
* "Why is my Rolls column showing a count for one group and a chain-modal toggle for another?"
* "Is this a bug?"

[OPT-263](https://linear.app/optionedge/issue/OPT-263/rolled-position-groups-display-is-confusing-strategy-label-rolls) improves the visuals; this issue adds the explanatory layer so the distinction feels intentional rather than surprising.

## Scope (candidates — to be refined during design)

* **Info popover/tooltip** on the Rolls column header and on the staged-roll sections, using the existing `InfoPopover.vue` pattern. One-sentence framing + a "learn more" link.
* **Inline explainer** inside the expanded card when a same-exp roll is shown — a short "Why one card?" hint that can be dismissed per-user (localStorage flag).
* **Expanded copy in** `RollChainModal.vue` — the modal already has a "learn more" section explaining the roll detection criteria; extend it to contrast with same-exp behavior so users see the full picture from either entry point.
* **Settings / Help page section** — a single canonical explainer page that both popovers link to, covering:
  * The mental model (position management vs new trade)
  * How detection works (same expiration → one group; different expiration → new group)
  * How rolls are counted (walk-and-balance; not `order_id`)
  * Why this matters for P&L reporting (each group is its own realized-P&L bucket)

## Non-goals

* No changes to the underlying data model or detection logic.
* Not a replacement for [OPT-263](https://linear.app/optionedge/issue/OPT-263/rolled-position-groups-display-is-confusing-strategy-label-rolls) — this is the documentation/affordance layer on top of it.

## Acceptance

* A new user looking at the Ledger for the first time can discover, without leaving the page, why their rolled positions render in two different shapes.
* Copy is consistent between the Rolls column popover, the roll-chain modal, and any Help page section (single source of truth).
* Dismissible inline explainer does not re-surface after dismissal.
