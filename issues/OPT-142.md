# OPT-142: Consolidate duplicate option legs on Positions page

## Problem

The Positions page shows one row per `PositionLot`. When the user opens the same option in multiple fills (e.g., two separate orders for -20 IBIT 260306C00039000 at different prices), two identical-looking rows appear. This level of detail belongs on the Ledger page.

## Solution

Consolidate in `src/routers/positions.py` when building `open_option_legs`. After the existing per-lot loop, merge legs that share the same consolidation key: `(symbol, strike, expiration, quantity_direction)`.

* `quantity`: Sum
* `cost_basis`: Sum
* `opening_price`: Recompute as `abs(cost_basis) / quantity / 100`
* `lot_id`: Replace with symbol (used as Alpine `:key`)

No frontend changes needed — all calculations already work with consolidated values.
