# OPT-137: Beta capacity gate — limit signups and show waitlist page when full

## Overview

Implement a capacity-limited free beta test. The first X users can sign up and use the app. Once the cap is reached, new visitors see a friendly "beta is full" page instead of the sign-up flow.

## Requirements

* Configurable max beta user count (e.g., `BETA_MAX_USERS=50` env var)
* Before allowing sign-up/onboarding, check current user count against the cap
* When full: redirect new users to a waitlist/closed-beta page that explains the beta is full and optionally collects their email for notification when spots open
* When open: normal sign-up flow proceeds as usual
* Existing users who already signed up are unaffected regardless of cap

## Considerations

* Where does the check happen? Supabase sign-up callback, or app-level after first login?
* Waitlist page design — friendly messaging, optional email collection
* Admin visibility — how many spots are taken vs remaining
* Future: ability to increase cap or notify waitlisted users"
