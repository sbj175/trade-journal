# OPT-297: Streaming underlying price — restore red treatment when the day's change is negative

`StreamingPrice.vue` renders the live underlying-price chip used on the Options and Equities pages. It currently shows green styling (`bg-tv-green/20 text-tv-green border-tv-green/50`) when the day's change is ≥ 0 and a neutral gray (`bg-tv-border text-tv-muted border-tv-border`) for everything else. The red counterpart that existed before the redesign got dropped.

Restore it: when `change < 0`, mirror the green styling exactly with the red token (`bg-tv-red/20 text-tv-red border-tv-red/50`), and keep the gray as the fallback for `change == 0` or null. Same treatment for the percent-change text next to the chip (`text-tv-red` when negative, otherwise the existing rules).

## Scope

`frontend/src/components/StreamingPrice.vue:14` and `:19` — replace the binary ternary with a three-way expression that distinguishes `> 0`, `< 0`, and `== 0`.

## Out of scope

Other live-price renderings (option leg prices, etc.) — only the underlying-price chip rendered by `StreamingPrice.vue` is in this change.
