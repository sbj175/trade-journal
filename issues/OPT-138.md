# OPT-138: Evaluate Supabase as hosting platform for limited beta

## Goal

Evaluate whether Supabase is the right platform to host OptionLedger for a limited beta (50–100 users), covering database, auth, hosting, and operational concerns.

## Context

Initial sizing analysis (OPT-136) shows \~5.5 MB per average user, meaning 100 users ≈ 550 MB. Supabase Pro ($25/mo) provides 8 GB database + 200 pooler clients, which comfortably covers this. We already use Supabase Auth. This research should go deeper into the full deployment picture.

## Areas to Evaluate

### 1\. Database (PostgreSQL)

* Confirm Pro tier (8 GB, Micro compute) is sufficient for 50–100 users
* Connection pooling strategy: Supavisor (shared, included) vs PgBouncer (dedicated, paid plans only) — which mode (session vs transaction) works best with SQLAlchemy's session-per-request pattern?
* Evaluate whether our current `pool_size=5, max_overflow=10` engine config needs adjustment for a hosted pooler
* Test Alembic migrations against a Supabase-hosted PostgreSQL instance
* Understand backup/restore options and point-in-time recovery on Pro tier

### 2\. Auth

* We already use Supabase Auth — confirm no changes needed for multi-user beta
* Evaluate onboarding flow: sign-up confirmation emails, password reset, etc.
* Check rate limits on auth endpoints for the Pro tier

### 3\. Application Hosting

* Supabase doesn't host application code — where does the FastAPI backend run?
* Evaluate options: [Fly.io](<http://Fly.io>), Railway, Render, or a small VPS (Hetzner, DigitalOcean)
* Consider latency between app server and Supabase DB (same region?)
* WebSocket support for live quotes — does the hosting platform support long-lived WS connections?

### 4\. Tastytrade OAuth Callback

* The OAuth redirect URI must be a publicly accessible URL — how does this work in a hosted environment?
* SSL/TLS certificate requirements for the callback endpoint

### 5\. Cost Projection

* Supabase Pro: $25/mo (includes $10 compute credit, Micro compute is effectively free)
* App hosting: estimate for FastAPI server (likely $5–20/mo depending on provider)
* Egress: Pro includes 250 GB/mo — estimate whether live quote streaming + API calls stay within this
* Total estimated monthly cost for 50–100 user beta

### 6\. Operational Concerns

* Monitoring: what dashboards/alerts does Supabase provide out of the box?
* How do we handle the admin dashboard (`admin_app.py`, port 8001) in production?
* Environment variable management for secrets (encryption key, Tastytrade client secret, etc.)
* Deployment pipeline: how do we ship updates? Docker image? Git push deploy?

### 7\. Migration Path

* What does it look like to move data from local Docker PostgreSQL to Supabase?
* Can we use `pg_dump`/`pg_restore` or do we need the migration script?
* Stamp Alembic version after migration

## Deliverable

A summary comment on this issue with a recommendation: go/no-go on Supabase for beta, recommended tier, estimated monthly cost, and any blockers or prerequisites to address first.
