# OPT-234: Fix broken import _recognize_from_lots in ledger_service reconciliation

## Bug

During sync, reconciliation auto-closes stale lots then calls `_refresh_group_status` which tries to import `_recognize_from_lots` from `src.pipeline.group_manager`. This function was renamed to `_label_from_all_lots` but the reference in `ledger_service.py` was not updated.

## Impact

Positions disappear after sync because the reconciliation error prevents proper group status refresh. The sync returns 200 but the groups are left in a broken state.

## Fix

Update import and call from `_recognize_from_lots` to `_label_from_all_lots` in `src/services/ledger_service.py`.
